Maintainer Notes

This documentation focuses on features and flows rather than file-by-file inventories. When adding new documentation:

  1. Start with the user perspective - What does the user see/do?
  2. Then explain the system - What happens under the hood?
  3. Map to implementation - Where is this code located?
  4. Include diagrams - Use Mermaid for flows and relationships
  5. Link everything - All references should be navigable links
