Endpoint: PUT /api/config/{key}

Overview

Property Value
Method PUT
Path /api/config/{key}
Auth public
Rate Limit N/A

What Does This Do?

Updates a configuration value in the persistent puppy.cfg file. This endpoint allows runtime modification of Code Puppy's behavior without restarting the application.

Request

Headers

Header Required Description
Content-Type yes application/json

Path Parameters

Parameter Type Description
key string The configuration key to update (must be a valid config key)

Request Body

{
  "value": <any>
}

Validation Rules

Field Rules
key Must exist in the list of valid config keys (see ListConfigKeys)
value Any JSON-serializable value; converted to string for storage

Response

Success Response (200)

{
  "key": "model",
  "value": "gpt-4o"
}

Error Responses

Status Code When
404 NOT_FOUND Config key does not exist in valid keys list
400 INVALID_REQUEST Request body is malformed or missing value field
500 INTERNAL_ERROR Failed to write to config file

Implementation Map

Layer Location Notes
Route definition code_puppy/api/routers/config.py Line 47-57
Handler code_puppy/api/routers/config.py:set_config_value()
Validation code_puppy/config.py:get_config_keys() Validates key exists
Persistence code_puppy/config.py:set_value() Writes to puppy.cfg

Flow

sequenceDiagram
    participant Client
    participant Router
    participant Handler
    participant Config
    
    Client->>Router: PUT /api/config/{key}
    Router->>Handler: set_config_value(key, value)
    Handler->>Config: get_config_keys()
    Config-->>Handler: valid_keys
    alt key not in valid_keys
        Handler-->>Client: 404 NOT_FOUND
    else key is valid
        Handler->>Config: set_value(key, str(value))
        Config-->>Handler: success
        Handler->>Config: get_value(key)
        Config-->>Handler: updated_value
        Handler-->>Client: 200 OK with updated value
    end

Examples

Example Request

curl -X PUT 'http://localhost:8000/api/config/model' \
  -H 'Content-Type: application/json' \
  -d '{"value": "gpt-4o"}'

Example Response

{
  "key": "model",
  "value": "gpt-4o"
}

Example Error Response (Invalid Key)

curl -X PUT 'http://localhost:8000/api/config/invalid_key' \
  -H 'Content-Type: application/json' \
  -d '{"value": "something"}'
{
  "detail": "Config key 'invalid_key' not found. Valid keys: [...]"
}

Edge Cases & Gotchas

WARNING

Values are always stored as strings - Even if you send a number or boolean, it will be converted to string via str(update.value) before persistence.

NOTE

No validation of value semantics - The API only validates that the key exists. It's up to the caller to ensure the value is appropriate for that key (e.g., sending "true" for boolean configs).
