Endpoint: GET /api/sessions/{id}/messages

Overview

Property Value
Method GET
Path /api/sessions/{id}/messages
Auth public
Rate Limit N/A

What Does This Do?

Retrieves the complete message history for a specific subagent session. This is the full conversation log including all user prompts, agent responses, tool calls, and tool results.

Headers

No specific headers required - -

Path Parameters

Parameter Type Description

None

None

Success Response (200 OK)

Message Structure

The message format depends on the pydantic-ai library's internal representation. Common message types include:

Role Content Type Description
user string or object User prompts or tool results
assistant object Agent responses, tool calls
system string System prompts (if present)

Data Transformations

Pickle to JSON Serialization

The _serialize_message() function converts pydantic-ai message objects to JSON-safe dictionaries:

def _serialize_message(msg: Any) -> Dict[str, Any]:
    # Try pydantic v2 model_dump
    if hasattr(msg, "model_dump"):
        return msg.model_dump(mode="json")
    # Fallback to __dict__
    elif hasattr(msg, "__dict__"):
        return {k: str(v) for k, v in msg.__dict__.items()}
    # Final fallback
    else:
        return {"content": str(msg)}

Edge Cases & Gotchas

WARNING

Pickle format: Messages are stored in Python pickle format, which is Python-specific. The serialization function attempts to make them JSON-safe, but complex objects may lose type information.

NOTE

Message ordering: Messages are returned in the order they were stored (chronological order of the conversation).

TIP

Large sessions: Sessions with many messages may take longer to deserialize. The 10-second timeout protects against hanging requests.
