Endpoint: POST /api/commands/autocomplete

Overview

Property Value
Method POST
Path /api/commands/autocomplete
Auth public
Rate Limit N/A

What Does This Do?

Provides intelligent autocomplete suggestions for slash commands as the user types. This endpoint powers the autocomplete functionality in the CLI, helping users discover available commands and their usage patterns.

Request

Headers

Header Required Description
Content-Type yes application/json

Request Body

{
  "partial": "string"  // Partial command string, e.g., "/se" or "/set mo"
}

Example Requests

// Empty input - get all commands
{
  "partial": ""
}

// Partial command name
{
  "partial": "/se"
}

// Complete command with args
{
  "partial": "/set mo"
}

Response

Success Response (200)

{
  "suggestions": [
    "/set",
    "/set_model",
    "/session"
  ]
}

Response Fields

Field Type Description
suggestions string[] List of autocomplete suggestions

Behavior Rules

Empty Input

When partial is empty or only whitespace:

  • Returns all available command names with leading /
  • Sorted alphabetically

Partial Command Name (No Space)

When input is just a partial command name like /se:

  • Returns matching command names and aliases
  • Case-insensitive matching
  • Includes both primary names and aliases

Complete Command with Args

When input includes a complete command name plus arguments like /set mo:

  • Returns usage hint as a single suggestion
  • Shows the expected argument format

Unknown Command

When the command name doesn't exist:

  • Returns empty suggestions array

Flow

sequenceDiagram
    participant Client
    participant API
    participant Registry
    
    Client->>API: POST /api/commands/autocomplete
    API->>API: Strip leading /
    alt Empty input
        API->>Registry: get_unique_commands()
        Registry-->>API: All commands
        API->>API: Format with /
        API-->>Client: All command names
    else Partial command name (no space)
        API->>Registry: get_unique_commands()
        Registry-->>API: All commands
        API->>API: Filter by prefix
        API->>Registry: Check aliases
        Registry-->>API: Matched aliases
        API-->>Client: Matching commands + aliases
    else Complete command with args
        API->>Registry: get_command(cmd_name)
        alt Command found
            Registry-->>API: Command object
            API->>API: Extract usage
            API-->>Client: Usage hint
        else Command not found
            API-->>Client: Empty suggestions
        end
    end

Examples

Example 1: Get All Commands

curl -X POST 'http://localhost:8000/api/commands/autocomplete' \
  -H 'Content-Type: application/json' \
  -d '{"partial": ""}'

Response:

{
  "suggestions": [
    "/add_model",
    "/agent",
    "/autosave",
    "/colors",
    "/commands",
    "/config",
    "/diff",
    "/help",
    "/hook",
    "/mcp",
    "/model",
    "/pin",
    "/quit",
    "/reset",
    "/schedule",
    "/session",
    "/set",
    "/skills",
    "/uc"
  ]
}

Example 2: Partial Command Match

curl -X POST 'http://localhost:8000/api/commands/autocomplete' \
  -H 'Content-Type: application/json' \
  -d '{"partial": "/se"}'

Response:

{
  "suggestions": [
    "/session",
    "/set",
    "/skills"
  ]
}

Example 3: Complete Command with Args

curl -X POST 'http://localhost:8000/api/commands/autocomplete' \
  -H 'Content-Type: application/json' \
  -d '{"partial": "/set model="}'

Response:

{
  "suggestions": [
    "/set <key>=<value>  # Set a configuration value"
  ]
}

Edge Cases & Gotchas

NOTE

Case-insensitive matching: Command names and aliases are matched case-insensitively, but suggestions are returned in their canonical form.

NOTE

Alias inclusion: Both primary command names and aliases are included in suggestions, so /config and /cfg might both appear.
