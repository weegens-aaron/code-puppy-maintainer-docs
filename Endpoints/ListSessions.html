Endpoint: GET /api/sessions

Overview

Property Value
Method GET
Path /api/sessions
Auth public
Rate Limit N/A

What Does This Do?

Retrieves a list of all available subagent sessions with their metadata. This endpoint is used by the system to display session history and allows debugging/inspection of past agent interactions.

Request

Headers

Header Required Description
No specific headers required - -

Path Parameters

None

Query Parameters

None

Request Body

None

Response

Success Response (200 OK)

Returns an array of session metadata objects:

[
  {
    "session_id": "abc123def456",
    "agent_name": "planning",
    "initial_prompt": "You are a planning agent...",
    "created_at": "2024-01-15T10:30:00Z",
    "last_updated": "2024-01-15T10:35:00Z",
    "message_count": 15
  },
  {
    "session_id": "xyz789",
    "agent_name": "code_reviewer",
    "initial_prompt": null,
    "created_at": "2024-01-14T08:00:00Z",
    "last_updated": "2024-01-14T08:10:00Z",
    "message_count": 5
  }
]

Response Fields

Field Type Description
session_id string Unique session identifier (filename without extension)
agent_name string \ null Name of the agent that created this session
initial_prompt string \ null The initial system prompt used for this session
created_at string \ null ISO timestamp when session was created
last_updated string \ null ISO timestamp of last message in session
message_count number Total number of messages in the session

Error Responses

Status When
200 Always succeeds - returns empty array if no sessions exist

Storage Format

Sessions are stored in DATA_DIR/subagent_sessions/ (typically ~/.code_puppy/data/subagent_sessions/):

File Purpose
{session_id}.txt JSON metadata (created_at, agent_name, etc.)
{session_id}.pkl Pickle file with full message history

Edge Cases & Gotchas

NOTE

Empty directory handling: If the sessions directory doesn't exist or is empty, returns [] (empty array) rather than an error.

WARNING

Timeout handling: File I/O runs in a thread pool with a 10-second timeout. If reading a metadata file times out, the session is still included with minimal info (just session_id).

TIP

Corrupted files: If a metadata file can't be parsed, the session is still included with basic info. This prevents one corrupted file from breaking the entire list.
