Endpoint: GET /api/agents

Overview

Property Value
Method GET
Path /api/agents
Auth public
Rate Limit N/A

What Does This Do?

Returns a list of all available agents registered in the Code Puppy system, including their names, display names, and descriptions. This endpoint is used by the web UI and external tools to discover what agents are available for selection.

Agent Discovery Process

The endpoint triggers agent discovery each time it's called:

  1. Registry Scan: Scans code_puppy/agents/ for Python agent classes
  2. JSON Agent Discovery: Loads JSON agent definitions from agents/*.json
  3. Filtering: Applies configuration filters (pack agents, UC agents)
  4. Instantiation: Creates temporary instances to extract metadata
  5. Caching: Results are cached per-call via message group ID

Agent Types

Type Source Example
Python Agent Python class agent_python_programmer.py
JSON Agent JSON file agents/custom-agent.json
Pack Agent Pack subdirectory agents/pack/bloodhound.py

Configuration

Feature Flags

Setting Purpose Default
pack_agents_enabled Enable/disable Pack agents true
universal_constructor_enabled Enable UC-dependent agents true

Agent Filtering

Agents are filtered based on configuration:

  • Pack agents (in PACK_AGENT_NAMES) are excluded if pack_agents_enabled=false
  • UC agents (in UC_AGENT_NAMES) are excluded if universal_constructor_enabled=false

Edge Cases & Gotchas

WARNING

Agent discovery runs on every call - This endpoint triggers a full agent scan each time, which may have performance implications if called frequently.

NOTE

Agent availability is dynamic - Newly created agents (via /api/commands or CLI) will appear immediately without server restart.

CAUTION

Filtering is configuration-dependent - The list returned may vary based on environment configuration (e.g., disabled features).
