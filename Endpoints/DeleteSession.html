Endpoint: DELETE /api/sessions/{id}

Overview

Property Value
Method DELETE
Path /api/sessions/{id}
Auth public
Rate Limit N/A

What Does This Do?

Permanently deletes a subagent session and all its associated data (metadata and message history). This is a destructive operation that cannot be undone.

Request

Headers

Header Required Description
No specific headers required - -

Path Parameters

Parameter Type Description
id string The session identifier to delete

Query Parameters

None

Request Body

None

Response

Success Response (200 OK)

{
  "message": "Session 'abc123def456' deleted"
}

Error Responses

Status Code When
404 NOT_FOUND Neither .txt nor .pkl file exists for this session

Edge Cases & Gotchas

WARNING

Destructive operation: This endpoint permanently deletes session data. There is no undo mechanism or trash/recycle bin.

NOTE

Partial deletion: If only one file exists (e.g., .txt but not .pkl), the endpoint still succeeds after deleting the existing file.

TIP

Idempotency: The endpoint is NOT idempotent - calling it twice will return 404 on the second call since the session no longer exists.

Safety Considerations

Before Deleting

Consider these questions:

  1. Do you need this session for debugging?
  2. Has the data been archived elsewhere?
  3. Is this a production system where audit trails matter?

Recommended Usage

  • Development/testing cleanup
  • Privacy/data retention compliance
  • Storage management
