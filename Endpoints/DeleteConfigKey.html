Endpoint: DELETE /api/config/{key}

Overview

Property Value
Method DELETE
Path /api/config/{key}
Auth public
Rate Limit N/A

What Does This Do?

Resets a configuration value to its default by removing it from the config file. After deletion, the system will use the built-in default value for that key.

Request

Path Parameters

Parameter Type Description
key string The configuration key to reset (e.g., default_agent, enable_dbos)

Headers

Header Required Description
Content-Type no application/json (not used for DELETE)

Response

Success Response (200)

{
  "message": "Config key 'default_agent' reset to default"
}

Error Responses

Status When
404 If the config key does not exist in the valid keys list
NOTE

Unlike GET/PUT which validate the key exists first, DELETE simply removes the key if present. If the key doesn't exist in the file, it succeeds silently (nothing to delete).

Examples

Example Request

curl -X DELETE 'http://localhost:8000/api/config/default_agent'

Example Response

{
  "message": "Config key 'default_agent' reset to default"
}

Reset Multiple Config Keys

# Reset agent configuration
curl -X DELETE 'http://localhost:8000/api/config/default_agent'

# Reset DBOS setting
curl -X DELETE 'http://localhost:8000/api/config/enable_dbos'

# Reset Universal Constructor
curl -X DELETE 'http://localhost:8000/api/config/enable_universal_constructor'

Edge Cases & Gotchas

WARNING

Silent success: If the key doesn't exist in the config file, the endpoint returns 200 OK without error. This is intentional - the end state (key has default value) is the same.

NOTE

Default values are code-defined: After deletion, the default value comes from the code (e.g., get_universal_constructor_enabled() returns True if key is missing). Check the config.py getter functions for defaults.
