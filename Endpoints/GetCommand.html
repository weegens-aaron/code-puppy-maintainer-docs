Endpoint: GET /api/commands/{name}

Overview

Property Value
Method GET
Path /api/commands/{name}
Auth public
Rate Limit N/A

What Does This Do?

Retrieves detailed information about a specific slash command by its name or alias. This endpoint is useful for frontend tools or external integrations that need to understand the capabilities, usage, and help text for individual commands.

Request

Path Parameters

Parameter Type Description
name string Command name or alias (without leading /)

Example Requests

# Get info about the 'set' command
curl -X GET 'http://localhost:8000/api/commands/set'

# Get info about an alias (e.g., 'ls' for 'list')
curl -X GET 'http://localhost:8000/api/commands/ls'

# Case-insensitive lookup
curl -X GET 'http://localhost:8000/api/commands/SET'

Response

Success Response (200 OK)

{
  "name": "set",
  "description": "Set a configuration value",
  "usage": "/set <key>=<value>",
  "aliases": ["config", "conf"],
  "category": "config",
  "detailed_help": "Sets a configuration key to a value. Use /list_configs to see available keys."
}

Response Fields

Field Type Description
name string Canonical command name
description string Short description of what the command does
usage string Usage pattern showing syntax
aliases array Alternative names for this command
category string Command category (e.g., "core", "config", "mcp")
detailed_help string or null Extended help text, if available

Error Responses

Status When
404 Command with given name/alias not found

404 Response Example

{
  "detail": "Command '/unknown' not found"
}

Command Lookup Logic

The endpoint performs a case-insensitive lookup that checks:

  1. Exact command name match - e.g., /set matches set
  2. Alias match - e.g., /ls matches list command
  3. Case normalization - e.g., /SET matches set

Lookup Priority

# Pseudocode for lookup logic
def get_command(name):
    name_lower = name.lower()
    
    # First, check direct command names
    for cmd in all_commands:
        if cmd.name.lower() == name_lower:
            return cmd
    
    # Then, check aliases
    for cmd in all_commands:
        if name_lower in [a.lower() for a in cmd.aliases]:
            return cmd
    
    return None

Edge Cases & Gotchas

WARNING

Alias collisions: If two commands have the same alias, the first registered command wins. This is rare but possible with custom plugins.

NOTE

No leading slash: The name parameter should NOT include the leading /. The API automatically handles the distinction between command names and their display format.

TIP

Use for dynamic help: Frontend tools can use this endpoint to build dynamic help menus without hardcoding command descriptions.
