Endpoint: GET /api/config/{key}

Overview

Property Value
Method GET
Path /api/config/{key}
Auth public
Rate Limit N/A

What Does This Do?

Retrieves a specific configuration value from the code_puppy configuration file (~/.code_puppy/puppy.cfg or XDG config location). This endpoint allows external tools and integrations to query individual configuration settings without exposing the entire config.

Request

Path Parameters

Parameter Type Description
key string The configuration key to retrieve (e.g., model, yolo_mode, enable_dbos)

Headers

Header Required Description
Content-Type no application/json (for response)

Response

Success Response (200 OK)

Returns the configuration value for the requested key.

{
  "key": "model",
  "value": "gpt-4o"
}

Response Schema:

Field Type Description
key string The requested configuration key
value any The current value of the config key (string, number, boolean, or null)

Error Responses

Status Code When
404 NOT_FOUND The requested config key does not exist

404 Error Response:

{
  "detail": "Config key 'invalid_key' not found. Valid keys: [\"allow_recursion\", \"auto_save_session\", ...]"
}

Implementation Map

Layer Location Notes
Route definition code_puppy/api/routers/config.py Line 33-42
Handler get_config_value() Validates key exists
Config access code_puppy/config.py get_value() and get_config_keys()

Flow

sequenceDiagram
    participant Client
    participant Router
    participant Handler
    participant Config
    
    Client->>Router: GET /api/config/{key}
    Router->>Handler: get_config_value(key)
    Handler->>Config: get_config_keys()
    Config-->>Handler: List of valid keys
    Handler->>Handler: Validate key in list
    alt Key exists
        Handler->>Config: get_value(key)
        Config-->>Handler: Config value
        Handler-->>Client: 200 OK + {key, value}
    else Key not found
        Handler-->>Client: 404 NOT_FOUND + error message
    end

Configuration Keys

The following keys are currently supported:

Core Settings

model string First in models.json Currently selected model

Edge Cases & Gotchas

WARNING

Key validation: The endpoint validates that the requested key exists in the combined set of: - Keys currently in puppy.cfg - Preset expected keys (e.g., yolo_mode, model, enable_dbos) Even if a key doesn't exist in the file yet, it will be recognized if it's in the preset list.

NOTE

Boolean values: Boolean config values are stored as strings ("true"/"false") in the config file but returned as-is. The API doesn't convert them to JSON booleans.

NOTE

Null values: If a key exists in the preset list but has no value in the config file, the endpoint will return null as the value.
