Endpoint: GET /api/sessions/{id}

Overview

Property Value
Method GET
Path /api/sessions/{id}
Auth public
Rate Limit N/A

What Does This Do?

Retrieves metadata for a specific subagent session by its ID. Returns the same information as the list endpoint but for a single session, including creation time, agent name, and message count.

Request

Headers

Header Required Description
No specific headers required - -

Path Parameters

Parameter Type Description
id string The session identifier (matches filename without .txt extension)

Query Parameters

None

Request Body

None

Response

Success Response (200 OK)

{
  "session_id": "abc123def456",
  "agent_name": "planning",
  "initial_prompt": "You are a planning agent...",
  "created_at": "2024-01-15T10:30:00Z",
  "last_updated": "2024-01-15T10:35:00Z",
  "message_count": 15
}

Response Fields

Field Type Description
session_id string The session identifier
agent_name string \ null Name of the agent that created this session
initial_prompt string \ null The initial system prompt used
created_at string \ null ISO timestamp of session creation
last_updated string \ null ISO timestamp of last update
message_count number Total messages in session

Error Responses

Status Code When
404 NOT_FOUND Session file does not exist
504 TIMEOUT File read operation exceeds 10-second timeout

Edge Cases & Gotchas

WARNING

Session vs Messages: This endpoint only returns metadata. To get the actual message content, use GET /api/sessions/{id}/messages.

NOTE

File format: The session metadata is stored as JSON in a .txt file (historical naming choice). The actual messages are in a separate .pkl (pickle) file.

TIP

Timeout behavior: Unlike the list endpoint which includes partial data on timeout, this endpoint returns a 504 error if the file read times out.
