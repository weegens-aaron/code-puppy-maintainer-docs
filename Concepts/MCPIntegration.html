Concept: MCP Integration

What Is This?

MCP (Model Context Protocol) Integration is the system that connects Code Puppy to external MCP servers. MCP servers provide specialized capabilities like file system access, database queries, browser automation, and more. The integration manages server lifecycle, handles errors, and exposes servers to AI agents as tools.

Why This Approach?

The MCP integration uses a manager pattern with managed servers to:

  • Standardize access: All servers follow the same interface
  • Handle lifecycle: Automatic startup, monitoring, and recovery
  • Provide resilience: Error isolation and retry mechanisms
  • Enable discovery: Server catalog and auto-completion
  • Support multiple types: stdio, SSE, and streamable HTTP servers

Example

from code_puppy.mcp_ import get_mcp_manager
from code_puppy.mcp_.managed_server import ServerConfig

# Get the manager
manager = get_mcp_manager()

# Register a server
config = ServerConfig(
    name="filesystem",
    type="stdio",
    config={
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-filesystem"]
    }
)
server_id = manager.register_server(config)

# Get servers for agent use
servers = manager.get_servers_for_agent()

Server Types

Stdio Servers

Run as child processes with stdio communication.

config = ServerConfig(
    name="filesystem",
    type="stdio",
    config={
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-filesystem"],
        "env": {}
    }
)

Characteristics:

  • Run as subprocess
  • Communicate via stdin/stdout
  • Easy to debug
  • Process lifecycle managed by Code Puppy

SSE Servers

Connect via Server-Sent Events over HTTP.

config = ServerConfig(
    name="remote-server",
    type="sse",
    config={
        "url": "http://localhost:3000/sse"
    }
)

Characteristics:

  • HTTP-based connection
  • Event-driven updates
  • Network-dependent
  • External process management

Streamable HTTP Servers

Use modern HTTP streaming.

config = ServerConfig(
    name="streaming-server",
    type="streamable_http",
    config={
        "url": "http://localhost:3000/mcp"
    }
)

Characteristics:

  • HTTP POST requests
  • Streaming responses
  • Modern protocol
  • Better performance

Server Lifecycle

States

stateDiagram-v2
    [*] --> Initialized
    Initialized --> Starting
    Starting --> Running
    Starting --> Failed
    Running --> Stopping
    Running --> Failed
    Stopping --> [*]
    Failed --> [*]
    
    Failed --> Starting : Retry
State Description
Initialized Server registered but not started
Starting Server is starting up
Running Server is operational
Stopping Server is shutting down
Failed Server failed to start or crashed
Quarantined Server repeatedly failed, disabled

Lifecycle Events

Event Trigger Action
on_start Server starting Log, update status
on_ready Server ready Mark as running
on_error Error occurred Log, potentially quarantine
on_stop Server stopping Cleanup resources

Server Registration

From Config File

Servers in mcp_servers.json are auto-registered:

{
  "filesystem": {
    "type": "stdio",
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem"],
    "enabled": true
  }
}

Programmatic Registration

from code_puppy.mcp_ import get_mcp_manager
from code_puppy.mcp_.managed_server import ServerConfig

manager = get_mcp_manager()

config = ServerConfig(
    name="my-server",
    type="stdio",
    config={"command": "python", "args": ["server.py"]}
)

server_id = manager.register_server(config)

Via Plugin

from code_puppy.callbacks import register_callback

def register_mcp_servers():
    return [
        {
            "name": "my-plugin-server",
            "config": ServerConfig(...)
        }
    ]

register_callback("register_mcp_servers", register_mcp_servers)

Agent Integration

Getting Servers for Agents

from code_puppy.mcp_ import get_mcp_manager

manager = get_mcp_manager()

# Get pydantic-ai compatible server instances
servers = manager.get_servers_for_agent()

# These can be passed to pydantic-ai Agent
agent = Agent("gpt-5.3-codex", mcp_servers=servers)

Server Filtering

# Get only enabled servers
enabled = manager.get_servers_for_agent(enabled_only=True)

# Get servers by name
specific = manager.get_servers_for_agent(names=["filesystem"])

Error Handling

Quarantine

Servers that repeatedly fail are quarantined:

# Server automatically quarantined after 3 failures
# Can be manually re-enabled
manager.unquarantine_server(server_id)

Error Isolation

Errors in one server don't affect others:

from code_puppy.mcp_.error_isolation import MCPErrorIsolator

isolator = MCPErrorIsolator()
isolator.isolate_server(server)

Retry Logic

Failed servers can be retried:

from code_puppy.mcp_.retry_manager import get_retry_manager

retry_manager = get_retry_manager()
retry_manager.schedule_retry(server_id)

Conventions & Rules

IMPORTANT

Server IDs: Each server has a unique ID (auto-generated if not provided)

IMPORTANT

Server names: Must be unique and match config file keys

IMPORTANT

Enabled state: Disabled servers are not started or exposed to agents

IMPORTANT

Error handling: Never let server errors crash the main application

Anti-Patterns to Avoid

CAUTION

Don't block on startup: Use async startup with timeouts

CAUTION

Don't ignore errors: Log all errors and track failure counts

CAUTION

Don't share servers: Each agent gets its own server instances

Extending This

To add a new server type:

  1. Create server class in code_puppy/mcp_/: ```python class MyCustomServer: async def connect(self): # Connection logic pass async def disconnect(self): # Cleanup logic pass ```
  1. Add to manager in manager.py: ```python def _create_server(self, config: ServerConfig): if config.type == "custom": return MyCustomServer(config) ```
  1. Document the type in this concept doc
