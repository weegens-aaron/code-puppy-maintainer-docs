Concept: Configuration Management

What Is This?

Configuration Management is the system that stores, retrieves, and manages Code Puppy's user settings. It uses a hierarchical approach with file-based configuration (puppy.cfg), environment variables, and runtime defaults to provide flexible, persistent settings across sessions.

Why This Approach?

The configuration system uses XDG Base Directory specification with INI file format to:

  • Follow standards: XDG paths are familiar to Linux/Unix users
  • Enable persistence: Settings survive across sessions
  • Support overrides: Environment variables can override file settings
  • Provide defaults: Sensible defaults when config is missing
  • Allow customization: Users can tune behavior via config file

Example

# Read a config value
from code_puppy.config import get_value

# Returns value from puppy.cfg or None
verbose = get_value("enable_streaming")

# Returns True/False with default
verbose = get_value("enable_streaming") or True

# Check boolean config
if str(verbose).lower() in ("1", "true", "yes", "on"):
    print("Streaming enabled")

Configuration Format

INI File Format

[puppy]
puppy_name = Code Puppy
owner_name = Developer
model = gpt-5.3-codex
enable_streaming = true
enable_universal_constructor = true
enable_pack_agents = false
compaction_strategy = summarize
protected_token_count = 100000
compaction_threshold = 120000

Boolean Values

Accepted values (case-insensitive):

  • True: 1, true, yes, on
  • False: 0, false, no, off, or missing

Reading Config

from code_puppy.config import get_value

# Get raw value (string or None)
model = get_value("model")

# Get boolean with default
streaming = get_value("enable_streaming")
if streaming is None:
    streaming = True  # Default
else:
    streaming = str(streaming).lower() in ("1", "true", "yes", "on")

# Helper function for booleans
from code_puppy.config import get_enable_streaming
streaming = get_enable_streaming()  # Returns True/False

Setting Config

from code_puppy.config import set_value

# Set a value
set_value("model", "gpt-5.3-codex")
set_value("enable_streaming", "true")

# Helper function for booleans
from code_puppy.config import set_universal_constructor_enabled
set_universal_constructor_enabled(True)

Conventions & Rules

IMPORTANT

Boolean format: Use lowercase true/false in config files

IMPORTANT

Section name: All keys must be under [puppy] section

IMPORTANT

Required keys: puppy_name and owner_name must exist

IMPORTANT

Case sensitivity: Keys are case-sensitive

Anti-Patterns to Avoid

CAUTION

Don't hardcode values: Always use get_value() or helper functions

CAUTION

Don't assume types: Config values are strings, convert as needed

CAUTION

Don't ignore None: get_value() returns None if key doesn't exist

Extending This

To add a new configuration key:

  1. Add to config file (optional, will be created if missing): ```ini [puppy] my_new_feature = true ```
  1. Create getter function in code_puppy/config.py: ```python def get_my_new_feature() -> bool: val = get_value("my_new_feature") if val is None: return True # Default return str(val).lower() in ("1", "true", "yes", "on") ```
  1. Create setter function (optional): ```python def set_my_new_feature(enabled: bool) -> None: set_value("my_new_feature", "true" if enabled else "false") ```
  1. Document the key in this concept doc
