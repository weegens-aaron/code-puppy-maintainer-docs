Concept: Agent Architecture

What Is This?

The Agent Architecture defines how Code Puppy creates, manages, and executes AI agents. An agent is a specialized AI assistant with specific capabilities, tools, and behaviors. The architecture supports both Python-based agents (hardcoded classes) and JSON-based agents (configurable files), all unified under a common BaseAgent interface.

Why This Approach?

The agent architecture uses a class-based registry pattern to:

  • Enable specialization: Different agents for different tasks (coding, reviewing, planning)
  • Support modularity: Plugins can register custom agents
  • Allow flexibility: JSON agents for quick configuration, Python agents for complex logic
  • Provide consistency: All agents share common interfaces and behaviors

Example: Python Agent

from .base_agent import BaseAgent

class MyCustomAgent(BaseAgent):
    """A specialized agent for code review."""
    
    def __init__(self):
        super().__init__()
        self.name = "Code Reviewer"
        self.description = "Specializes in reviewing code for bugs and style"
    
    def __get_tools__(self):
        """Return list of tool names this agent can use."""
        return [
            "list_files",
            "read_file",
            "edit_file",
            "agent_run_shell_command",
        ]
    
    def __get_system_prompt__(self):
        """Return the system prompt for this agent."""
        return """You are a code review expert. Focus on:
        - Bug detection
        - Code style
        - Security issues
        - Performance optimizations"""

Agent Types

Python Agents

Hardcoded Python classes that inherit from BaseAgent.

Characteristics:

  • Full access to all Code Puppy APIs
  • Can implement custom logic
  • Require code changes to modify
  • Compiled and tested

Examples:

Agent File Purpose
PromptReviewerAgent prompt_reviewer.py Review prompts
WatchdogAgent pack/watchdog.py Monitor system
HuskyAgent pack/husky.py General purpose

JSON Agents

Configuration files that define agent properties.

Characteristics:

  • No code changes needed
  • Easy to create and modify
  • Limited to predefined options
  • Stored in ~/.code_puppy/agents/

Example Location:

~/.code_puppy/agents/
  my-custom-agent.json
  python-tutor.json

JSON Agents

def _load_json_agents():
    """Load JSON agent configurations."""
    agents_dir = Path(CONFIG_DIR) / "agents"
    if not agents_dir.exists():
        return []
    
    agents = []
    for json_file in agents_dir.glob("*.json"):
        with open(json_file) as f:
            config = json.load(f)
            agents.append(JSONAgent(config))

Plugin Agents

Registered via register_agents callback:

from code_puppy.callbacks import register_callback

def register_my_agents():
    return [
        {"name": "my-agent", "class": MyAgentClass},
    ]

register_callback("register_agents", register_my_agents)

Agent Configuration

Model Selection

Agents can pin specific models:

def __get_model_name__(self):
    """Pin this agent to a specific model."""
    return "gpt-5.3-codex"

Or use global model:

def __get_model_name__(self):
    """Use the globally configured model."""
    return None  # Falls back to global model

Tool Selection

Agents define their available tools:

def __get_tools__(self):
    """Return tools this agent can use."""
    return [
        "list_files",
        "read_file",
        "edit_file",
    ]

Agents define their behavior:

def __get_system_prompt__(self):

    Always provide clear explanations.
    Include code examples when helpful."""

Conventions & Rules

IMPORTANT

Naming: Agent class names use PascalCase (e.g., MyCustomAgent)

IMPORTANT

File naming: Agent files use snake_case (e.g., my_custom_agent.py)

IMPORTANT

Inheritance: All agents must inherit from BaseAgent

IMPORTANT

Tool names: Use snake_case and match registry names

Anti-Patterns to Avoid

CAUTION

Don't bypass BaseAgent: Always inherit from BaseAgent for compatibility

CAUTION

Don't hardcode tools: Use __get_tools__() for flexibility

CAUTION

Don't ignore errors: Handle tool execution errors gracefully
