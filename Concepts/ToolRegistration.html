Concept: Tool Registration

What Is This?

Tool registration is the mechanism by which Code Puppy makes capabilities available to AI agents. Tools are functions that agents can call to perform actions like reading files, running commands, controlling browsers, or interacting with external systems. The registration system supports both built-in tools and plugin-provided tools through a unified interface.

Why This Approach?

The tool registration system uses a registry pattern with lazy loading to:

  • Support modularity: Plugins can add tools without modifying core code
  • Enable dynamic discovery: Tools are loaded when needed, not at startup
  • Allow conditional registration: Tools can be enabled/disabled based on configuration
  • Provide type safety: Each tool has a well-defined schema for agent consumption

How It Works

The Pattern

  1. Tool Definition: A tool is a function with a specific signature and metadata
  2. Registration Function: Each tool has a register_*() function that registers it with an agent
  3. Registry: The TOOL_REGISTRY dictionary maps tool names to registration functions
  4. Agent Registration: When an agent is created, specific tools are registered for it
# Tool definition
def my_tool(context: RunContext, param1: str, param2: int) -> str:
    """Tool description.
    
    Args:
        context: Agent context
        param1: First parameter
        param2: Second parameter
    
    Returns:
        Result description
    """
    # Tool implementation
    return "result"

# Registration function
def register_my_tool(agent):
    """Register the tool with an agent."""
    agent.add_tool(my_tool)

Example

# Built-in tool registration
TOOL_REGISTRY = {
    "list_files": register_list_files,
    "read_file": register_read_file,
    "edit_file": register_edit_file,
}

# Plugin tool registration
def register_plugin_tools():
    return [
        {
            "name": "my_plugin_tool",
            "register_func": register_my_plugin_tool,
        }
    ]

Tool Expansion

NOTE

Backward compatibility: Some tools expand into multiple tools for better granularity.

TOOL_EXPANSIONS = {
    "edit_file": ["create_file", "replace_in_file", "delete_snippet"],
}

When an agent requests edit_file, the system registers all three expanded tools instead.

Plugin Tool Registration

Plugins register tools via the register_tools callback:

from code_puppy.callbacks import register_callback

def register_plugin_tools():
    return [
        {
            "name": "my_custom_tool",
            "register_func": register_my_custom_tool,
        }
    ]

register_callback("register_tools", register_plugin_tools)

Conditional Registration

Tools can be conditionally registered based on:

  1. Model type: Some tools only work with specific models
  2. Configuration: Tools can be enabled/disabled via config
  3. Extended thinking: agent_share_your_reasoning is skipped when extended thinking is active
def register_tools_for_agent(agent, tool_names, model_name=None):
    # Skip reasoning tool when extended thinking is active
    if tool_name == "agent_share_your_reasoning" and has_extended_thinking_active(model_name):
        continue
    
    # Skip UC tools if disabled
    if tool_name == "universal_constructor" and not get_universal_constructor_enabled():
        continue
    
    # Register the tool
    register_func = TOOL_REGISTRY[tool_name]
    register_func(agent)

Conventions & Rules

IMPORTANT

Naming: Tool names use snake_case (e.g., run_shell_command, not runShellCommand)

IMPORTANT

Registration functions: Must be named register_<tool_name>() and accept an agent parameter

IMPORTANT

Tool signatures: Must use RunContext as first parameter for pydantic-ai compatibility

Anti-Patterns to Avoid

CAUTION

Don't register tools at module load: Use lazy loading via _load_plugin_tools()

CAUTION

Don't modify TOOL_REGISTRY directly: Use the registration function pattern

CAUTION

Don't ignore registration errors: Log warnings for unknown tools

Extending This

To add a new tool:

  1. Define the tool function in a new or existing tools module
  2. Create a registration function register_<tool_name>(agent)
  3. Add to TOOL_REGISTRY in code_puppy/tools/__init__.py
  4. Update agent configuration to include the tool name
# 1. Tool definition
def my_new_tool(context: RunContext, param: str) -> str:
    """Do something useful."""
    return "result"

# 2. Registration function
def register_my_new_tool(agent):
    agent.add_tool(my_new_tool)

# 3. Add to registry
TOOL_REGISTRY["my_new_tool"] = register_my_new_tool

# 4. Include in agent config
agent_config = {
    "tools": ["my_new_tool", ...],
}
