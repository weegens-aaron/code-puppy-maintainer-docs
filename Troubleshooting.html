Common Issues

Agent Issues

Symptom: Agent not responding

Likely Causes:

  1. LLM API key not configured
  2. Model not found or invalid
  3. Rate limit exceeded

Diagnosis:

# Check current model
/model

# Check API keys in environment
echo $OPENAI_API_KEY
echo $ANTHROPIC_API_KEY

Resolution:

  1. Set missing API keys as environment variables
  2. Use /add_model to configure a working model
  3. Wait for rate limits to reset

Symptom: Agent crashes mid-task

Likely Causes:

  1. Token limit exceeded
  2. Invalid tool response
  3. Network timeout

Diagnosis:

# Check logs for error
tail -f ~/.code_puppy/logs/*.log

Resolution:

  1. Use /truncate to reduce message history
  2. Retry the task
  3. Check network connectivity

Model Issues

Symptom: Model not found

Likely Causes:

  1. Model name typo
  2. Model not configured
  3. Provider API down

Resolution:

  1. Use /add_model to browse available models
  2. Check ~/.code_puppy/extra_models.json
  3. Verify provider status

Symptom: Model returns errors

Likely Causes:

  1. Invalid API key
  2. Insufficient credits/quota
  3. Model deprecated

Resolution:

  1. Regenerate API key
  2. Check provider billing
  3. Switch to alternative model

MCP Issues

Symptom: MCP server won't start

Likely Causes:

  1. Server binary not found
  2. Configuration invalid
  3. Port already in use

Diagnosis:

# Check server logs
~/.code_puppy/mcp_logs/<server-name>.log

Resolution:

  1. Install missing server dependencies
  2. Fix configuration in ~/.code_puppy/mcp_servers.json
  3. Use different port

Symptom: MCP tools not available

Likely Causes:

  1. Server not started
  2. Tools not registered
  3. Connection lost

Resolution:

  1. Start server: /mcp start <server>
  2. Check server status: /mcp status
  3. Restart server if needed

Plugin Issues

Symptom: Plugin not loading

Likely Causes:

  1. register_callbacks.py missing
  2. Callback registration error
  3. Plugin in wrong location

Resolution:

  1. Ensure plugin has register_callbacks.py at root
  2. Check for syntax errors
  3. Place in ~/.code_puppy/plugins/<name>/

Symptom: Custom command not working

Likely Causes:

  1. Command not registered
  2. Hook not triggered
  3. Plugin conflict

Resolution:

  1. Check plugin logs for errors
  2. Verify callback registration
  3. Disable other plugins to isolate

Browser Issues

Symptom: Browser automation fails

Likely Causes:

  1. Chromium not installed
  2. Display issues (headless)
  3. Website blocking automation

Resolution:

  1. Install Chromium: uv run playwright install chromium
  2. Check for display errors in logs
  3. Try different website

Emergency Procedures

Agent Frozen/Unresponsive

  1. Immediate action: Press Ctrl+C to cancel
  2. Diagnostic step: Check logs for stuck operation
  3. Resolution step: Restart Code Puppy
  4. Verification step: Run simple task to confirm

Data Corruption

  1. Immediate action: Stop using Code Puppy
  2. Diagnostic step: Backup ~/.code_puppy/
  3. Resolution step: Delete config, reconfigure
  4. Verification step: Test with simple task

Plugin Causing Crashes

  1. Immediate action: Disable plugin by renaming folder
  2. Diagnostic step: Check which plugin loaded last
  3. Resolution step: Fix plugin code or remove
  4. Verification step: Restart Code Puppy
