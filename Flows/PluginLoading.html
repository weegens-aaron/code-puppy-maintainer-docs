Flow: Plugin Loading

What Happens?

When Code Puppy starts up, it automatically discovers and loads all available plugins from both built-in and user-provided locations. Each plugin registers callbacks that hook into various phases of the application lifecycle, enabling extensibility without modifying core code.

Trigger

  • Application startup: Plugins are loaded once during CLI initialization in cli_runner.py
  • Idempotent: Subsequent calls to load plugins are ignored to prevent duplicate registrations

Outcome

  • All plugin callbacks are registered in the global callback registry
  • Plugins can intercept and modify application behavior at defined hook points
  • Built-in and user plugins are tracked separately for debugging

1. Guard Check

  • What: Check if _PLUGINS_LOADED flag is already True
  • Where: code_puppy/plugins/__init__.py:load_plugin_callbacks()
  • Failure mode: If skipped, plugins would be reloaded causing duplicate callback registrations

2. Load Built-in Plugins

  • What: Iterate through code_puppy/plugins/ subdirectories
  • Where: code_puppy/plugins/__init__.py:_load_builtin_plugins()
  • Failure mode: Import errors are caught and logged; other plugins continue loading

3. Special Case Handling

  • What: shell_safety plugin is conditionally loaded based on safety_permission_level config
  • Where: code_puppy/plugins/__init__.py:_load_builtin_plugins() lines 30-37
  • Failure mode: If safety level is "high" or "medium", shell_safety is skipped silently

4. Load User Plugins

  • What: Iterate through ~/.code_puppy/plugins/ subdirectories
  • Where: code_puppy/plugins/__init__.py:_load_user_plugins()
  • Failure mode: User plugin errors are logged but don't crash the application

5. Register Callbacks

  • What: Plugin's register_callbacks.py executes at import time, calling register_callback() for each hook
  • Where: Module scope of each plugin's register_callbacks.py
  • Failure mode: Callback registration errors are caught by the callback engine's error handling

6. Mark as Loaded

  • What: Set _PLUGINS_LOADED = True to prevent future reloads
  • Where: code_puppy/plugins/__init__.py:load_plugin_callbacks() line 167
  • Failure mode: If not set, subsequent imports would cause duplicate registrations

Data Transformations

Built-in scan plugins_dir: Path List of plugin names _load_builtin_plugins()
User scan user_plugins_dir: Path List of plugin names _load_user_plugins()
Final result builtin list, user list {"builtin": [...], "user": [...]} load_plugin_callbacks()

Failure Handling

Failure Point Detection Recovery Alert?
Plugin import error ImportError exception Log warning, continue to next plugin Yes (warning)
Plugin runtime error General Exception Log error with traceback, continue Yes (error)
Callback registration error ValueError/TypeError in register_callback() Propagates to plugin loader, logged Yes (error)
User plugins dir missing Path.exists() returns False Return empty list, no error No
shell_safety blocked Config check fails Skip plugin silently Debug log only

Performance Characteristics

  • Typical duration: 50-200ms depending on plugin count
  • Bottlenecks: Python module import overhead for each plugin
  • Scaling considerations: Plugin count should remain under ~50 for acceptable startup time

Debugging This Flow

Key Log Messages

Message Meaning Action
Loaded plugins: builtin=[...], user=[...] Successful load summary Check if expected plugins are listed
Skipping shell_safety plugin - safety_permission_level is 'X' Plugin blocked by config Set safety_permission_level=low or none if needed
Failed to import callbacks from plugin X: ... Import error Check plugin's register_callbacks.py for syntax errors
Unexpected error loading plugin X: ... Runtime error Check plugin code for exceptions at import time
Plugins already loaded, skipping duplicate load Idempotency guard fired Normal - indicates multiple load attempts

Common Issues

Plugin not loading

Symptom: Expected plugin missing from loaded list

Likely Causes:

  1. Missing register_callbacks.py file
  2. Import error in the plugin code
  3. For shell_safety: safety level too high

Diagnosis:

# Check plugin structure
ls -la code_puppy/plugins/my_plugin/

# Run with debug logging
python -m code_puppy --log-level DEBUG

Resolution:

  1. Ensure register_callbacks.py exists in plugin directory
  2. Check for syntax errors: python -c "from code_puppy.plugins.my_plugin import register_callbacks"
  3. For shell_safety: set safety_permission_level=low in config

Duplicate callback registrations

Symptom: Callbacks firing multiple times

Likely Cause:

  • _PLUGINS_LOADED flag not working (shouldn't happen in normal operation)

Resolution:

  • Restart the application - the flag is module-level and resets on restart
