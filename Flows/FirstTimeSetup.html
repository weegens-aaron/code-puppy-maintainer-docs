Flow: First-Time Setup

What Happens?

When a user runs Code Puppy for the first time, the system guides them through a 5-slide interactive tutorial, optionally sets up OAuth authentication, and creates their initial configuration files.

Trigger

  • User action: Running code-puppy or code-puppy -i for the first time
  • Condition: No onboarding complete marker file exists at ~/.code_puppy/onboarding_complete
  • Override: Set CODE_PUPPY_SKIP_TUTORIAL=1 to skip automatically

Outcome

  • Configuration directory created at ~/.code_puppy/ (or XDG paths if env vars set)
  • puppy.cfg created with user's name and puppy name
  • Onboarding marker file created
  • Optional OAuth authentication completed if user selected a model provider
  • User enters interactive mode ready to code

Step-by-Step

flowchart TD
    A[Start: code-puppy] --> B{Config Exists?}
    B -->|No| C[Create XDG Directories]
    B -->|Yes| D{Onboarding Complete?}
    C --> D
    D -->|No| E[Show Onboarding Wizard]
    D -->|Yes| J[Skip Tutorial]
    E --> F{User Selected Model?}
    F -->|ChatGPT| G[Run ChatGPT OAuth]
    F -->|Claude| H[Run Claude OAuth]
    F -->|API Keys| I[Skip OAuth]
    F -->|Skip| I
    G --> K[Mark Onboarding Complete]
    H --> K
    I --> K
    J --> L[Enter Interactive Mode]
    K --> L

1. Directory & Config Initialization

  • What: Create XDG directories and ensure puppy.cfg exists
  • Where: code_puppy/config.py → ensure_config_exists()
  • Failure mode: If directory creation fails, app exits with error

Creates these directories (0700 permissions):

  • ~/.code_puppy/ (config)
  • ~/.code_puppy/data/ (models, agents, skills)
  • ~/.code_puppy/cache/ (autosaves)
  • ~/.code_puppy/state/ (command history, DBOS)

Prompts for required config values:

  • puppy_name: What to call the puppy
  • owner_name: Owner's name

2. Onboarding Check

  • What: Determine if tutorial should be shown
  • Where: code_puppy/command_line/onboarding_wizard.py → should_show_onboarding()
  • Failure mode: None - defaults to showing if marker doesn't exist

Checks:

  1. CODE_PUPPY_SKIP_TUTORIAL env var (skip if "1", "true", "yes")
  2. ~/.code_puppy/onboarding_complete marker file (skip if exists)

3. Interactive Tutorial (5 Slides)

  • What: Show 5-slide TUI tutorial
  • Where: code_puppy/command_line/onboarding_wizard.py → run_onboarding_wizard()
  • Failure mode: Caught and logged as warning, app continues

Slide 1: Welcome

  • Quick intro to setup steps
  • Progress indicator: ● ○ ○ ○ ○

Slide 2: Model Selection

  • User picks model provider:
  • ChatGPT Plus/Pro/Max (OAuth)
  • Claude Code Pro/Max (OAuth)
  • API Keys (manual config)
  • OpenRouter (single key)
  • Skip for now
  • Selection triggers OAuth flow or skips

Slide 3: MCP Servers (Optional)

  • Explains MCP server integration
  • Shows /mcp install and /mcp list commands
  • User can skip

Slide 4: Agent Usage Guide

  • When to use Code Puppy (default) vs Planning Agent
  • Examples of appropriate tasks for each

Slide 5: Ready to Roll

  • Essential commands reminder
  • Shows which OAuth flow will run (if any)
  • Press Enter to start

Navigation:

  • →/l: Next slide
  • ←/h: Previous slide
  • ↑↓/jk: Navigate options
  • Enter: Select/confirm
  • ESC: Skip tutorial

4. OAuth Authentication (Conditional)

  • What: Complete OAuth flow for selected provider
  • Where:
  • ChatGPT: code_puppy/plugins/chatgpt_oauth/oauth_flow.py → run_oauth_flow()
  • Claude: code_puppy/plugins/claude_code_oauth/register_callbacks.py → _perform_authentication()
  • Failure mode: OAuth errors logged, app continues without model configured

ChatGPT OAuth:

  1. Opens browser for authentication
  2. Exchanges token for model access
  3. Sets model to chatgpt-gpt-5.3-codex

Claude OAuth:

  1. Opens browser for authentication
  2. Exchanges token for model access
  3. Sets model to claude-code-claude-opus-4-6

5. Mark Onboarding Complete

  • What: Create marker file to prevent re-showing tutorial
  • Where: code_puppy/command_line/onboarding_wizard.py → mark_onboarding_complete()
  • Failure mode: None - warning logged if fails

Creates ~/.code_puppy/onboarding_complete with content "completed"

6. Enter Interactive Mode

  • What: Start the main interactive CLI loop
  • Where: code_puppy/cli_runner.py → interactive_mode()
  • Failure mode: None - this is the main app loop

Shows:

  • Exit instructions
  • Command hints
  • MOTD (Message of the Day)
  • Truecolor warning (if applicable)
  • Ready prompt

Data Transformations

Stage Input Output Location
Config init None puppy.cfg with user/puppy names config.py:ensure_config_exists()
Tutorial result User selections result string (chatgpt/claude/completed/skipped) onboarding_wizard.py:run_onboarding_wizard()
OAuth OAuth result Model name set in config cli_runner.py (after wizard)
Marker None onboarding_complete file onboarding_wizard.py:mark_onboarding_complete()

Failure Handling

Failure Point Detection Recovery Alert?
Directory creation OSError Exit with error message Yes (error)
Tutorial crash Exception caught Log warning, skip tutorial Yes (warning)
OAuth failure Exception in OAuth flow Log error, continue without model Yes (error)
Marker file write OSError Log warning, may show tutorial again Yes (warning)

Performance Characteristics

  • Typical duration: 1-2 minutes (including tutorial reading time)
  • OAuth duration: 30-60 seconds (browser auth + token exchange)
  • Bottlenecks: None - all synchronous, no network calls except OAuth
  • Scaling considerations: N/A - single user setup

Common Issues

Symptom: Tutorial doesn't show on first run

Likely Causes:

  1. CODE_PUPPY_SKIP_TUTORIAL=1 env var is set
  2. Marker file already exists from previous run

Diagnosis:

# Check env var
echo $CODE_PUPPY_SKIP_TUTORIAL

# Check marker file
ls -la ~/.code_puppy/onboarding_complete

Resolution:

# Remove marker to re-show tutorial
rm ~/.code_puppy/onboarding_complete

# Or run tutorial manually
code-puppy -i
# Then type /tutorial

Symptom: OAuth flow hangs

Likely Causes:

  1. Browser didn't open
  2. Callback server port blocked
  3. Network issues

Diagnosis:

  • Check if browser opened
  • Check terminal for error messages
  • Verify network connectivity

Resolution:

  1. Cancel with Ctrl+C
  2. Manually set API key: /set OPENAI_API_KEY=sk-...
  3. Or use /add_model to configure manually

Symptom: Config prompts don't appear

Likely Causes:

  1. Config file already exists
  2. stdin not available (non-interactive run)

Diagnosis:

cat ~/.code_puppy/puppy.cfg

Resolution:

  • If file exists, config is already set
  • If file missing, run interactively: code-puppy -i
