Flow: Message History Processing

What Happens?

When an agent receives new messages during execution, the message history processing flow manages the conversation history to prevent token limit exhaustion while preserving context. This includes deduplication, compaction (summarization or truncation), and filtering of invalid messages.

Trigger

  • When: Every time the agent receives new messages from pydantic-ai
  • Where: message_history_accumulator hook in base_agent.py
  • Context: During agent execution, after model response but before next turn

Outcome

  • Deduplicated message history with no duplicate messages
  • Token usage managed via compaction when threshold exceeded
  • Invalid messages (empty thinking parts, interrupted tool calls) filtered out
  • Message history always ends with a ModelRequest (user prompt)
  • Callbacks fired for observability and plugin extension

Step-by-Step

    B --> C[Fire on_message_history_processor_start]
    C --> D[Calculate Message Hashes]
    D --> E{Duplicate?}
    E -->|Yes| F[Skip Unless Last Message]
    E -->|No| G[Add to History]
    F --> H
    G --> H[Call message_history_processor]
    H --> I[Calculate Token Usage]
    I --> J{Over Threshold?}
    J -->|No| K[Filter Empty Thinking Parts]
    J -->|Yes| L{Strategy?}
    L -->|Truncation| M[Truncate Recent Messages]
    L -->|Summarization| N{Pending Tool Calls?}
    N -->|Yes| O[Defer Compaction]
    N -->|No| P[Summarize Older Messages]
    M --> Q[Track Dropped Hashes]
    P --> Q
    O --> R[Return Original Messages]
    Q --> K
    K --> S[Ensure History Ends with Request]
    S --> T[Fire on_message_history_processor_end]
    T --> U[Return Processed History]

1. Message Accumulation

  • What: New messages from pydantic-ai are received and deduplicated
  • Where: message_history_accumulator in code_puppy/agents/base_agent.py:1560-1604
  • Failure mode: If deduplication fails, duplicate messages could accumulate

Deduplication Logic:

message_history_hashes = set([self.hash_message(m) for m in _message_history])
for i, msg in enumerate(messages):
    msg_hash = self.hash_message(msg)
    if msg_hash not in message_history_hashes:
        # Always preserve the last message (user's new prompt) even if duplicate
        if i == last_msg_index or msg_hash not in self.get_compacted_message_hashes():
            _message_history.append(msg)

Special Rules:

  • Last message (user prompt) is always added, even if hash matches
  • This prevents "ghost task" bug where short prompts like "yes" collide with compacted messages

2. Token Usage Calculation

  • What: Calculate current token usage and compare to model limit
  • Where: message_history_processor in code_puppy/agents/base_agent.py:1035-1105
  • Failure mode: Incorrect token estimation could trigger premature compaction

Token Calculation:

message_tokens = sum(self.estimate_tokens_for_message(msg) for msg in messages)
context_overhead = self.estimate_context_overhead_tokens()
total_current_tokens = message_tokens + context_overhead
proportion_used = total_current_tokens / model_max

3. Compaction Decision

  • What: Determine if compaction is needed based on threshold
  • Where: message_history_processor in code_puppy/agents/base_agent.py:1055-1065
  • Failure mode: If threshold misconfigured, could compaction too aggressively or not at all

Configuration:

Setting Purpose Default
compaction_threshold When to trigger compaction (0.0-1.0) 0.8 (80%)
compaction_strategy How to compact: summarization or truncation summarization
protected_tokens Tokens to protect from truncation 4000

4. Pending Tool Call Protection

  • What: Check for incomplete tool calls before summarization
  • Where: message_history_processor in code_puppy/agents/base_agent.py:1065-1075
  • Failure mode: Summarizing mid-tool-call would break tool execution

Race Condition Protection:

if compaction_strategy == "summarization" and self.has_pending_tool_calls(messages):
    pending_count = self.get_pending_tool_call_count(messages)
    emit_warning(f"⚠️  Summarization deferred: {pending_count} pending tool call(s)")
    self.request_delayed_compaction()
    return messages  # Return original, defer compaction

5. Compaction Execution

  • What: Apply configured compaction strategy
  • Where: code_puppy/agents/base_agent.py:1075-1095
  • Failure mode: Summarization LLM call could fail

Truncation Strategy:

if compaction_strategy == "truncation":
    protected_tokens = get_protected_token_count()
    filtered_messages = self.filter_huge_messages(messages)
    result_messages = self.truncation(filtered_messages, protected_tokens)
    # Track dropped hashes to prevent re-injection

Summarization Strategy:

else:
    result_messages, summarized_messages = self.summarize_messages(
        self.filter_huge_messages(messages)
    )
    # Add summarized message hashes to compacted set
    for m in summarized_messages:
        self.add_compacted_message_hash(self.hash_message(m))

6. Message Filtering

  • What: Remove invalid message structures
  • Where: message_history_accumulator in code_puppy/agents/base_agent.py:1604-1625
  • Failure mode: Invalid messages could cause API errors

Filtering Rules:

  • Single-part messages with empty ThinkingPart → removed entirely
  • Multi-part messages with empty ThinkingPart → strip empty part, keep message
  • Ensure history ends with ModelRequest (not ModelResponse)

7. Callback Notification

  • What: Fire end callback with processing statistics
  • Where: message_history_accumulator in code_puppy/agents/base_agent.py:1638-1648
  • Failure mode: Plugin errors could crash agent (should be caught by plugin engine)

Callback Signature:

on_message_history_processor_end(
    agent_name=self.name,
    session_id=getattr(self, "session_id", None),
    message_history=list(final_history),
    messages_added=messages_added,
    messages_filtered=messages_filtered,
)

Failure Handling

Failure Point Detection Recovery Alert?
Summarization LLM error SummarizationError exception Return original messages (no compaction) Yes (emit_error)
Token estimation error Exception in estimate_tokens_for_message Skip compaction, continue Yes (emit_error)
Pending tool calls has_pending_tool_calls() returns True Defer compaction, set flag Yes (emit_warning)
Invalid message structure Empty ThinkingPart detected Filter out during accumulation No (silent)

Performance Characteristics

  • Typical duration: <100ms for deduplication + filtering
  • Summarization duration: 5-30 seconds (depends on message count and LLM latency)
  • Bottlenecks: Summarization LLM call (if triggered)
  • Scaling considerations: More messages = longer summarization time

Common Issues

Issue: Ghost Tasks After Compaction

Symptom: Agent re-runs same task after compaction Likely Cause: Short user prompts (e.g., "yes") hash-collide with compacted messages Fix: Last message is always preserved (already implemented in v2.0+)

Issue: Summarization Fails Repeatedly

Symptom: SummarizationError in logs, compaction doesn't happen Likely Cause: Summarization model unavailable or context too large Fix: Switch to truncation: /set compaction_strategy=truncation

Issue: Agent Loses Context Too Aggressively

Symptom: Agent forgets important details after compaction Likely Cause: compaction_threshold too low or protected_tokens too small Fix: Increase threshold: /set compaction_threshold=0.9 or increase protected tokens
