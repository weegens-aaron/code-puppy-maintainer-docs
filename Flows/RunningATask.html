Flow: Running a Task

What Happens?

This flow describes how Code Puppy processes a user's coding task from initial input through agent execution and response delivery. It covers the complete journey from typing a command to receiving the agent's response with full support for attachments, cancellation, and MCP tool integration.

Trigger

A user task is initiated by:

  • Interactive mode: User types a prompt at the CLI and presses Enter
  • Single prompt mode: code-puppy -p "your task" command line flag
  • Initial command: code-puppy "your task" (deprecated but supported)

Outcome

The agent processes the task, executes any necessary tool calls (file operations, shell commands, MCP tools), and returns a structured response that is streamed back to the user with proper markdown formatting.

Step-by-Step

flowchart TD
    A[User Input] --> B{Shell Passthrough?}
    B -->|Yes: !cmd| C[Execute Directly]
    B -->|No| D[Parse Attachments]
    D --> E[Get Clipboard Images]
    E --> F[Build Prompt Payload]
    F --> G{Has Attachments?}
    G -->|Yes| H[Combine with Text]
    G -->|No| I[Text Only]
    H --> J[Create Agent Task]
    I --> J
    J --> K[Fire agent_run_start Hook]
    K --> L[Run Agent with MCP]
    L --> M{Execution Status}
    M -->|Success| N[Stream Response]
    M -->|Cancelled| O[Cleanup & Return]
    M -->|Error| P[Log & Emit Error]
    N --> Q[Emit Response Message]
    Q --> R[Update Message History]
    R --> S[Return to User]

1. Input Processing

  • What: User types a task or prompt
  • Where: code_puppy/cli_runner.py:interactive_mode()
  • Failure mode: Invalid input is passed through; empty prompts after attachment removal trigger a warning

2. Shell Passthrough Check

  • What: Detect if input starts with ! for direct shell execution
  • Where: code_puppy/command_line/shell_passthrough.py
  • Failure mode: N/A - bypasses agent entirely

3. Attachment Parsing

  • What: Extract file paths, URLs, and clipboard images from the prompt
  • Where: code_puppy/command_line/attachments.py:parse_prompt_attachments()
  • Failure mode: Invalid file paths generate warnings but don't block execution

4. Clipboard Image Retrieval

  • What: Get pending clipboard images and clear the pending queue
  • Where: code_puppy/command_line/clipboard.py:get_clipboard_manager()
  • Failure mode: No clipboard images - continues with text only

5. Agent Task Creation

  • What: Create an async task for the agent to run
  • Where: code_puppy/cli_runner.py:run_prompt_with_attachments()
  • Failure mode: Task creation fails if agent is in invalid state

6. Agent Run Start Hook

  • What: Fire agent_run_start callback for plugins
  • Where: code_puppy/agents/base_agent.py:run_with_mcp()
  • Failure mode: Hook failures are caught and logged - don't block agent execution

7. Agent Execution with MCP

  • What: Run the pydantic-ai agent with MCP tools, attachments, and usage limits
  • Where: code_puppy/agents/base_agent.py:run_with_mcp()
  • Failure mode:
  • UsageLimitExceeded - user can ask to continue
  • McpError - malfunctioning MCP server
  • CancelledError - user cancelled via Ctrl+K/Ctrl+C

8. Response Streaming

  • What: Stream the agent's response through the event handler
  • Where: code_puppy/agents/event_stream_handler.py
  • Failure mode: Stream errors are logged but don't crash the agent

9. Message History Update

  • What: Update the agent's message history with the complete conversation
  • Where: code_puppy/cli_runner.py:interactive_mode()
  • Failure mode: History corruption could affect future turns

Failure Handling

Failure Point Detection Recovery Alert?
Invalid attachment Warning emitted Continue without it Yes (warning)
Usage limit reached UsageLimitExceeded exception User can ask to continue Yes (info)
MCP server error McpError exception Disable MCP server Yes (info)
User cancellation CancelledError exception Clean shutdown Yes (info)
Unexpected error Exception group Log to file Yes (error)

Performance Characteristics

  • Typical duration: 5-60 seconds for simple tasks, minutes for complex code generation
  • Bottlenecks: LLM response time, tool call execution (file I/O, shell commands)
  • Scaling considerations: Concurrent subagent tasks can multiply resource usage

Debugging This Flow

Key Log Messages

Message Meaning Action
Attachments detected -> files: N, clipboard images: N Prompt has attachments Verify paths are correct
🔄 Attempting delayed compaction Token limit approaching Check context size settings
Usage limit exceeded Request limit reached Ask agent to continue
Agent task cancelled User pressed Ctrl+K Normal cancellation

Common Issues

Symptom Likely Cause Fix
Prompt appears empty All text was attachment paths Add instructions before paths
Agent doesn't respond MCP server blocking Check MCP status with /mcp status
Terminal becomes unresponsive Long-running shell command Press Ctrl+X to cancel command
Response not streaming Console conflict Check for other Live displays
