Feature: Skill Usage

What Does This Do?

Skill usage enables Code Puppy's AI agents to discover, activate, and leverage installed skills to accomplish user tasks. When a skill is available, agents can automatically detect it, load its instructions, and use its capabilities as part of their workflow.

Why Does This Exist?

  • Augmented Capabilities: Skills extend what agents can do without hardcoding every capability
  • Context-Aware Assistance: Agents match user requests to relevant skills automatically
  • Modular Design: Skills are self-contained with their own documentation and resources
  • User Control: Users can enable/disable specific skills based on their needs

How It Works

User Perspective

  1. User asks agent to do something (e.g., "analyze this data file")
  2. Agent sees available skills in system prompt
  3. Agent calls activate_skill(skill_name) to load full instructions
  4. Agent follows skill's guidance to complete the task
  5. User sees skill being used in agent's reasoning

System Perspective

    participant User

The Skill Lifecycle

1. Discovery Phase (App Startup)

# In register_callbacks.py
register_callback("get_model_system_prompt", _inject_skills_into_prompt)

When the system prompt is built:

  1. Check if skills are enabled
  2. Scan configured directories
  3. Parse SKILL.md metadata for each
  4. Filter out disabled skills
  5. Build XML listing

2. Prompt Injection

<available_skills>
  <skill>
    <name>data-exploration</name>
    <description>Analyze and explore CSV/JSON data files</description>
  </skill>
  <skill>
    <name>browser-control</name>
    <description>Automate browser interactions</description>
  </skill>
</available_skills>

# Agent Skills

When `<available_skills>` appears in context, match user tasks to skill descriptions.
Call `activate_skill(skill_name)` to load full instructions before starting the task.
Use `list_or_search_skills(query)` to search for relevant skills.

3. Skill Activation (During Agent Run)

# Agent decides to use a skill
agent: "I'll use the data-exploration skill for this"
agent: activate_skill("data-exploration")

# Tool handler loads full content
content = load_full_skill_content(skill_path)
resources = get_skill_resources(skill_path)

# Content injected into conversation
# Agent now has full instructions

4. Skill Execution

The full SKILL.md content typically includes:

  • Purpose: What the skill does
  • Guidance: Step-by-step instructions
  • Resources: Associated files (scripts, configs, etc.)
  • Examples: Usage patterns

Skill Metadata Format

Skills define metadata in SKILL.md frontmatter:

---
name: data-exploration
description: Analyze and explore CSV/JSON data files
version: 1.0
author: Code Puppy
tags:
  - data
  - analysis
  - csv
  - json
---

Edge Cases & Gotchas

WARNING

Skills must have SKILL.md - Directories without SKILL.md are discovered but not included in prompts.

NOTE

Disabled skills are hidden - Skills in disabled_skills config are completely excluded from agent context.

CAUTION

XML escaping - Skill descriptions are XML-escaped to prevent prompt injection.

TIP

Use tags for discoverability - Agents search by name, description, AND tags.

Error Scenarios

Scenario Behavior Agent Sees
Skill not found Returns error "Skill 'X' not found. Use list_or_search_skills to see available skills."
Skills disabled Returns error "Skills integration is disabled. Enable it with /set skills_enabled=true"
SKILL.md missing Returns error "Failed to load content for skill 'X'"
Directory unreadable Skips silently Skill not included in list

Usage Examples

Agent Automatically Uses a Skill

User: "Can you analyze this sales.csv file?"

Agent: "I'll use the data-exploration skill for this.
        activate_skill('data-exploration')"

# Skill content loaded, agent follows guidance
Agent: "Reading sales.csv and computing summary statistics..."

Manual Skill Search

User: "What skills do I have for working with databases?"

Agent: list_or_search_skills("database")

# Returns matching skills
[
  {name: "sql-helper", description: "Generate and optimize SQL queries"},
  {name: "postgres-admin", description: "PostgreSQL administration tasks"}
]

Disable a Skill

> /skills disable browser-control

✓ Skill 'browser-control' has been disabled.

Testing This Feature

Manual Test

  1. Install a skill: /skills install
  2. Start a new agent session
  3. Ask the agent to do something the skill handles
  4. Verify agent calls activate_skill()
  5. Check that skill content is loaded

Verify Skill is in Prompt

# Check system prompt contains <available_skills>
# Look for skill name and description

Test Skill Activation

# Call activate_skill directly
result = activate_skill("data-exploration")
assert result.content != ""
assert "SKILL.md" content in result.content
