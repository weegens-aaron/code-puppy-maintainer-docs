Feature: Task Scheduling

What Does This Do?

Task Scheduling allows you to define automated prompts that run periodically in the background using a daemon process. You can schedule Code Puppy to perform recurring tasks like daily code reviews, periodic dependency checks, or regular status reports without manual intervention.

Why Does This Exist?

Many development workflows benefit from automation:

  • Daily standup prep: Auto-generate progress summaries
  • Code hygiene: Periodic linting, security scans, or dependency updates
  • Monitoring: Regular health checks or usage reports
  • Routine tasks: Recurring code reviews or documentation updates

Instead of manually running the same prompts repeatedly, you define them once and let the scheduler handle execution.

How It Works

User Perspective

  1. Run /scheduler in the Code Puppy CLI
  2. Browse existing scheduled tasks (or create new ones)
  3. Configure task parameters: name, schedule, agent, model, prompt
  4. Start the background daemon
  5. Tasks run automatically at their scheduled intervals
  6. Monitor execution via logs and status indicators

Configuration

Schedule Types

Type Value Format Example
interval <number><unit> where unit is s, m, h, d 30m, 2h, 1d
hourly 1h Runs every hour
daily 24h Runs once per day
cron Not yet supported Would be 0 9 * * *

Task Parameters

Parameter Description Required
name Human-readable task name Yes
prompt The prompt to execute Yes
agent Agent to use (e.g., code-puppy) No (default: code-puppy)
model Model override No (uses default)
schedule_type interval, hourly, daily, cron Yes
schedule_value Schedule value (e.g., 1h) Yes
working_directory Where to run the task No (default: current dir)

File Locations

File Purpose
~/.code_puppy/data/scheduled_tasks.json Task definitions
~/.code_puppy/data/scheduler.pid Daemon PID tracking
~/.code_puppy/data/scheduler_logs/<id>.log Task execution logs

Edge Cases & Gotchas

WARNING

Cron schedules are not yet supported - The system accepts cron as a schedule type but logs a warning and skips execution. Only interval, hourly, and daily work currently.

NOTE

Daemon must be running - Scheduled tasks only execute when the daemon is active. Use s in the scheduler menu to start/stop it.

TIP

Working directory validation - If the configured working directory doesn't exist, the task fails immediately with exit code -1.

CAUTION

TOCTOU race condition - There's a small window where two instances could start simultaneously. The daemon re-checks after starting to mitigate this.

Error Scenarios

Scenario Behavior User Sees
Working directory missing Task fails, status = failed Exit code -1 in log
code-puppy not in PATH Task fails, status = failed "code-puppy not found" error
Daemon not running Tasks don't execute Daemon status shows "STOPPED"
JSON config corrupted Tasks not loaded Empty task list, no error
Task prompt empty Task created but may fail Depends on CLI behavior

Testing This Feature

Manual Testing Steps

# 1. Start Code Puppy
code-puppy

# 2. Open scheduler menu
/scheduler

# 3. Create a test task:
#    - Press 'n' for new task
#    - Name: "Test Task"
#    - Schedule: "Every hour"
#    - Agent: "code-puppy"
#    - Prompt: "echo 'Hello from scheduled task'"
#    - Working directory: "."

# 4. Start the daemon:
#    - Press 's' to toggle daemon on

# 5. Verify daemon is running:
#    - Status should show "🐕 Daemon: RUNNING (PID <number>)"

# 6. Check task execution:
#    - Wait for scheduled time (or press 'r' to run immediately)
#    - Press 't' to tail the log file

# 7. Stop the daemon:
#    - Press 's' again

Quick Test (Immediate Execution)

# Create a task with immediate interval
# Then press 'r' in the scheduler menu to run it now
# Check the log file: ~/.code_puppy/data/scheduler_logs/<task-id>.log
