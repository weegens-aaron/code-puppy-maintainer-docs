Feature: MCP Server Management

What Does This Do?

MCP Server Management provides a complete system for registering, starting, stopping, and monitoring Model Context Protocol (MCP) servers that extend Code Puppy's capabilities with external tools and data sources.

Why Does This Exist?

MCP servers allow Code Puppy to interact with external systems like file systems, databases, APIs, and custom tools. This feature provides:

  • Centralized management of all MCP server connections
  • Runtime control (start/stop/restart servers without restarting Code Puppy)
  • Health monitoring and automatic quarantine of failing servers
  • Interactive CLI for managing servers via /mcp commands
  • Catalog integration for discovering and installing pre-configured servers

How It Works

User Perspective

Users interact with MCP servers through the /mcp command family:

/mcp list              # Show all registered servers and their status
/mcp start <name>      # Enable and start a server
/mcp stop <name>       # Disable and stop a server
/mcp status <name>     # Show detailed status for a server
/mcp install           # Browse and install from catalog
/mcp edit <name>       # Edit server configuration
/mcp remove <name>     # Remove a server
/mcp logs <name>       # View server logs
/mcp test <name>       # Test server connection

Server Types

Stdio Servers

  • Communication: stdin/stdout pipes
  • Use Case: Local processes (file systems, CLI tools)
  • Implementation: BlockingMCPServerStdio
  • Config Example: ```json { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem"], "timeout": 60 } ```

SSE Servers

  • Communication: Server-Sent Events over HTTP
  • Use Case: Remote servers, web services
  • Implementation: MCPServerSSE
  • Config Example: ```json { "url": "http://localhost:8080/sse", "timeout": 30 } ```

HTTP Servers

  • Communication: Streamable HTTP
  • Use Case: Modern HTTP-based MCP servers
  • Implementation: MCPServerStreamableHTTP
  • Config Example: ```json { "url": "http://localhost:8080/mcp", "headers": {"Authorization": "Bearer token"} } ```

Server States

State Description User Action
STOPPED Registered but not enabled Run /mcp start
STARTING Starting subprocess/connection Wait for ready
RUNNING Active and available Normal operation
STOPPING Shutting down Wait for stop
ERROR Failed to start/connect Check logs, restart
QUARANTINED Temporarily disabled due to errors Wait or manually unquarantine

Configuration

Server Configuration File

Servers are stored in ~/.local/share/code_puppy/mcp_registry.json:

{
  "uuid-123": {
    "id": "uuid-123",
    "name": "filesystem",
    "type": "stdio",
    "enabled": true,
    "config": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/dir"]
    }
  }
}

Environment Variables

Server configs support $VAR and ${VAR} expansion:

{
  "command": "${MCP_FILESYSTEM_CMD}",
  "args": ["${MCP_FILESYSTEM_ARGS}"]
}

Edge Cases & Gotchas

WARNING

Async Start Behavior: Stdio servers start their subprocess asynchronously. /mcp start returns immediately - use /mcp status to check if fully ready.

WARNING

Server Must Be Enabled: Servers are created in disabled state. They must be explicitly started with /mcp start before agents can use them.

NOTE

Quarantine Auto-Expiry: Quarantined servers automatically recover after the quarantine period expires.

NOTE

Name Uniqueness: Server names must be unique. Use IDs internally, names for CLI.

Error Scenarios

Scenario Behavior User Sees
Server not found Command fails gracefully "Server 'X' not found" with suggestions
Invalid config Registration rejected Validation error with details
Subprocess fails State → ERROR Error message in status
Connection timeout State → ERROR (SSE/HTTP) Timeout error in status
Catalog unavailable Install falls back to menu "Catalog not available" message

Testing This Feature

Manual Testing

# Start Code Puppy
code-puppy

# List servers (should be empty initially)
/mcp list

# Install a server from catalog
/mcp install

# Start the filesystem server
/mcp start filesystem

# Check status
/mcp status filesystem

# Test the server
/mcp test filesystem

# Stop the server
/mcp stop filesystem

# Remove the server
/mcp remove filesystem

Programmatic Testing

from code_puppy.mcp_.manager import get_mcp_manager

manager = get_mcp_manager()

# List all servers
servers = manager.list_servers()

# Start a server
success = await manager.start_server("filesystem")

# Get server status
status = manager.get_server_status("filesystem")
