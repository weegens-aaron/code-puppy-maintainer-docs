Feature: Browser Control

What Does This Do?

Browser Control provides terminal-based web automation capabilities, allowing agents to navigate websites, interact with page elements, extract content, and take screenshots - all through a headless Chromium browser powered by Playwright. This enables agents to perform web-based tasks like form filling, data extraction, testing web applications, and visual analysis of web pages.

Why Does This Exist?

Many real-world tasks require web interaction:

  • Testing web applications - Verify UI behavior, check for regressions
  • Data extraction - Scrape information from websites that lack APIs
  • Form automation - Fill out online forms, submit applications
  • Visual validation - Take screenshots for comparison or analysis
  • Web research - Navigate and gather information from multiple sites

The browser tools give agents the ability to see and interact with the web just like a human would, but faster and more reliably.

How It Works

User Perspective

An agent can:

  1. Initialize a browser session
  2. Navigate to any URL
  3. Find elements by role, text, label, or selector
  4. Click, type, select options, check boxes
  5. Take screenshots for visual analysis
  6. Manage multiple pages/tabs
  7. Save and reuse automation workflows

Browser Session Management

Session Isolation

Each agent session gets its own browser instance with isolated state:

  • Separate profile directory under XDG_CACHE_HOME/code_puppy/browser_profiles/<session_id>/
  • Independent cookies, localStorage, session storage
  • Supports parallel agent execution without interference

Context Propagation

# Context variable for browser session
_browser_session_var: contextvars.ContextVar[Optional[str]]

# Set before agent execution
token = set_browser_session(session_id)
try:
    # Agent runs with this session
    ...
finally:
    reset_browser_session(token)

Cleanup

  • Automatic cleanup on shutdown via atexit handler
  • State saved to storage_state.json before close
  • Persistent profiles allow resuming sessions

Edge Cases & Gotchas

WARNING

Element selectors must be valid - Invalid selectors cause failures. Prefer semantic locators (role, label, text) over CSS/XPath when possible.

WARNING

Headless mode is default - Set BROWSER_HEADLESS=false to see the browser window for debugging.

NOTE

Screenshots use BinaryContent - The browser_screenshot_analyze tool returns images as BinaryContent so multimodal models can directly analyze them.

NOTE

Profiles persist across runs - Browser state (cookies, localStorage) is saved and restored. Clear profile directory to reset.

CAUTION

Timeout defaults to 10 seconds - Slow pages may need timeout parameter increased.

Error Scenarios

Scenario Behavior User Sees
Browser not initialized Tool returns error dict "error": "Browser not initialized"
Element not found Wait times out "error": "Timeout 10000ms exceeded"
Navigation fails Returns error with URL "error": "net::ERR_NAME_NOT_RESOLVED"
Invalid selector Playwright throws "error": "Error: Strict mode violation"

Testing This Feature

Manual Testing

# Set visible browser for debugging
export BROWSER_HEADLESS=false

# Run code_puppy
uv run code_puppy

# In agent, try:
browser_initialize(headless=false)
browser_navigate(url="https://example.com")
browser_screenshot_analyze()

Tool Testing

# Run browser tool tests
pytest tests/tools/browser/ -v
