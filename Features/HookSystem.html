Feature: Hook System

What Does This Do?

The Hook System is Code Puppy's extensibility engine — it allows plugins to intercept, observe, and modify virtually any operation the system performs. From file operations to agent execution, tool calls to model creation, hooks let you inject custom behavior without touching core code.

Think of it as event-driven programming for AI agents: when something happens (an event), your plugin's callback function runs.

Why Does This Exist?

Code Puppy is designed to be highly customizable without requiring core modifications. The Hook System enables:

  • Safe extensibility: Plugins add features without risking core stability
  • User customization: Users can tailor behavior to their workflow
  • Modular architecture: Features are isolated in plugins under code_puppy/plugins/
  • Testing isolation: Each hook can be tested independently
  • Graceful degradation: If a plugin fails, the core system keeps running

How It Works

User Perspective

As a user or plugin developer, you:

  1. Create a plugin directory under code_puppy/plugins/ or ~/.code_puppy/plugins/
  2. **Add a register_callbacks.py file** with your hook functions
  3. **Call register_callback("<hook_name>", your_function)** at module scope
  4. Code Puppy auto-discovers and loads your plugin on startup
# code_puppy/plugins/my_plugin/register_callbacks.py
from code_puppy.callbacks import register_callback

def on_shell_command(context, command, cwd=None, timeout=60):
    """Block dangerous commands."""
    if "rm -rf /" in command:
        return {"blocked": True}
    return None  # Don't block

register_callback("run_shell_command", on_shell_command)

System Perspective

When an event occurs:

  1. Core code calls the appropriate on_<event>() function (e.g., on_run_shell_command())
  2. All registered callbacks for that hook are invoked sequentially
  3. Return values are collected — some hooks allow blocking/short-circuiting
  4. Errors are logged but don't crash the system (fail gracefully)

Configuration

Hooks are configured programmatically in register_callbacks.py files. No separate config file needed.

Plugin Structure

code_puppy/plugins/
└── my_plugin/
    ├── __init__.py          # Optional: plugin metadata
    └── register_callbacks.py # Required: hook registrations

Registration Pattern

from code_puppy.callbacks import register_callback

def my_callback(arg1, arg2):
    # Your logic here
    return result  # Optional: return value depends on hook type

register_callback("hook_name", my_callback)

Edge Cases & Gotchas

WARNING

Duplicate registrations — If a plugin is loaded multiple times, the same callback may register twice. The system detects this and logs a debug message, skipping the duplicate.

WARNING

Blocking hooks — Some hooks (like file_permission, run_shell_command) can block operations. Returning False or {"blocked": True} cancels the operation.

NOTE

Sync vs Async — Hooks can be sync or async. The system handles both: sync hooks run directly, async hooks are awaited. If you're in an async context and return a coroutine, it will be awaited.

NOTE

Error isolation — If a callback raises an exception, it's logged but doesn't crash the system. Other callbacks still run.

TIP

**Return None for "not handled"** — For hooks like custom_command, return None if you don't own that command, allowing other plugins to handle it.

Error Scenarios

Scenario Behavior User Sees
Callback raises exception Logged as error, other callbacks continue Error in logs, operation may proceed
Duplicate callback registration Skipped with debug log Nothing (silent)
Invalid hook name ValueError at registration time Error message listing valid hooks
Blocking hook returns False Operation cancelled "Permission denied" or similar message

Testing This Feature

Manual Testing

  1. Create a test plugin in code_puppy/plugins/test_hooks/register_callbacks.py
  2. Add hooks that print messages or log events
  3. Run Code Puppy and trigger the events
  4. Check logs for your callback output
# Test plugin example
from code_puppy.callbacks import register_callback

def test_startup():
    print("=== TEST: Startup hook fired! ===")

register_callback("startup", test_startup)

Programmatic Testing

The hook engine (code_puppy/hook_engine/) has comprehensive tests in tests/hook_engine/:

# Run hook engine tests
pytest tests/hook_engine/ -v

Advanced: Hook Engine

For Claude Code-compatible hooks with pattern matching, use the advanced hook engine:

from code_puppy.hook_engine import HookEngine, EventData

config = {
    "PreToolUse": [{
        "matcher": "Bash|agent_run_shell_command",
        "hooks": [{
            "type": "command",
            "command": "bash .claude/hooks/my-check.sh",
            "timeout": 5000
        }]
    }]
}

engine = HookEngine(config)
event_data = EventData(
    event_type="PreToolUse",
    tool_name="agent_run_shell_command",
    tool_args={"command": "git status"}
)

result = await engine.process_event("PreToolUse", event_data)
if result.blocked:
    print(f"Blocked: {result.blocking_reason}")

See code_puppy/hook_engine/README.md for full documentation.
