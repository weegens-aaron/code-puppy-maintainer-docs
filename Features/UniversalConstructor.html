Feature: Universal Constructor

What Does This Do?

The Universal Constructor (UC) is a dynamic tool creation system that lets you create, manage, and execute custom Python tools at runtime. Instead of waiting for pre-built skills, you can write your own tools on-the-fly and have Code Puppy execute them immediately.

Think of it as "skills on demand" - you describe what you need in Python, and Code Puppy creates a callable tool that persists across sessions.

Why Does This Exist?

  • Rapid Prototyping: Test new tool ideas without going through the full skills installation process
  • Custom Workflows: Create tools specific to your projects that don't exist in the public catalog
  • Learning & Experimentation: Understand how tools work by building your own
  • Session-Specific Tools: Create temporary tools for one-off tasks

How It Works

User Perspective

  1. Create a tool: Provide Python code with a function and optional metadata
  2. List available tools: See all created tools and their status
  3. Call a tool: Execute any enabled tool by name with arguments
  4. Update tools: Modify existing tool code
  5. Get info: View tool details and source code

Tool Structure

Every UC tool is a Python file with this structure:

# Required: Tool metadata
TOOL_META = {
    "name": "my_tool",
    "description": "What my tool does",
    "enabled": True,
    "version": "1.0.0",
}

# Required: Callable function (name should match TOOL_META.name)
def my_tool(arg1: str, arg2: int = 10) -> str:
    """Docstring explaining what the tool does."""
    return f"Result: {arg1}, {arg2}"

Namespacing

Tools can be organized into namespaces using dot notation:

  • weather → weather.py
  • api.weather → api/weather.py
  • api.finance.stocks → api/finance/stocks.py

Actions

list - List All Tools

await universal_constructor(action="list")

Returns all enabled tools with metadata, signatures, and source paths.

call - Execute a Tool

await universal_constructor(
    action="call",
    tool_name="my_tool",
    tool_args={"arg1": "value", "arg2": 42}
)

Executes the tool with a 30-second timeout. Returns result or error.

create - Create a New Tool

await universal_constructor(
    action="create",
    tool_name="my_tool",
    python_code="def my_tool(): return 'hello'",
    description="A simple greeting tool"
)

Creates a new tool file, validates it, and reloads the registry.

update - Modify an Existing Tool

await universal_constructor(
    action="update",
    tool_name="my_tool",
    python_code="def my_tool(): return 'updated'"
)

Replaces existing tool code. Tool must already exist.

info - Get Tool Details

await universal_constructor(
    action="info",
    tool_name="my_tool"
)

Returns tool metadata, source code, and signature.

Edge Cases & Gotchas

WARNING

Tool naming: The function name in your code should match the TOOL_META.name field. If they don't match, the tool may not be callable as expected.

NOTE

Auto-generated TOOL_META: If you don't include TOOL_META in your code, one will be auto-generated from the tool_name and description parameters.

CAUTION

Dangerous patterns: The sandbox checks for dangerous imports (subprocess, eval, exec, etc.) and will warn you, but won't block execution. Review warnings before trusting tools.

TIP

Code formatting: New tools are automatically formatted with ruff. If ruff is not installed, you'll get a warning but the tool will still be created.

Error Scenarios

Scenario Behavior User Sees
Syntax error in code Creation fails "Syntax error in code: ..."
Tool not found Call fails "Tool 'X' not found"
Tool disabled Call fails "Tool 'X' is disabled"
Invalid arguments Call fails "Invalid arguments for 'X': ..."
Tool timeout (>30s) Call fails "Tool 'X' timed out after 30s"
Missing TOOL_META Creation succeeds with warning "TOOL_META was auto-generated"

Testing This Feature

Manual Testing

  1. Enable UC (if disabled): ``` /config set enable_universal_constructor true ```
  1. Create a simple tool: ``` /uc create tool_name=hello python_code="def hello(): return 'Hello, World!'" ```
  1. List tools: ``` /uc list ```
  1. Call the tool: ``` /uc call tool_name=hello ```
  1. Verify file was created: ``` ls ~/.code_puppy/plugins/universal_constructor/ ```

Example Tool Creation Flow

# In an agent session, ask Code Puppy to create a tool:
"Create a tool called 'add_numbers' that takes two numbers and returns their sum"

# Code Puppy will generate code like:
TOOL_META = {
    "name": "add_numbers",
    "description": "Add two numbers together",
    "enabled": True,
    "version": "1.0.0",
}

def add_numbers(a: int, b: int) -> int:
    """Add two numbers and return the sum."""
    return a + b

# Then call it:
"Call add_numbers with 5 and 3"
