Feature: Agent Switching

What Does This Do?

Agent switching allows users to change which AI agent is active in their current terminal session. Each agent has specialized capabilities (code review, planning, security auditing, etc.), and switching lets you adapt the AI assistant to different tasks without restarting the application.

Why Does This Exist?

Different tasks require different AI expertise. A code reviewer agent has different tools and prompts than a planning agent. Rather than running multiple instances or restarting, users can seamlessly switch between specialized agents while preserving conversation history for each.

How It Works

User Perspective

  1. User invokes /agent command or clicks agent picker in the UI
  2. A split-panel interface shows all available agents
  3. User navigates with arrow keys and presses Enter to select
  4. The active agent reloads with the new agent's configuration
  5. Previous agent's conversation history is saved and restored when switched back

Edge Cases & Gotchas

WARNING

Pinned models override global model changes - If an agent has a pinned model, switching the global model won't affect it until unpinned.

WARNING

Cannot delete the active agent - Must switch away from an agent before deleting its clone.

NOTE

Session isolation - Each terminal session maintains its own active agent via PPID-based session IDs. Multiple terminal windows can use different agents simultaneously.

NOTE

History preservation - Each agent's conversation history is saved in memory and restored when switched back, maintaining context across switches.

JSON Agents

Users can create custom agents in ~/.code_puppy/agents/ as JSON files. These are discovered automatically and appear alongside built-in agents.

Clone Agents

When cloning an agent via the picker (C key), a JSON copy is created with -clone-N suffix. Clones can be customized independently and deleted.

Error Scenarios

Scenario Behavior User Sees
Agent not found Falls back to code-puppy Warning about agent not found
JSON agent parse error Agent skipped in registry Warning about JSON parsing failure
Session file corrupted Starts fresh with default agent No error, silent recovery
Dead session cleanup Removes orphaned session entries No user-visible effect
