Feature: Terminal Integration

What Does This Do?

Terminal Integration provides Code Puppy with the ability to manage terminal state across platforms, run interactive shell sessions, and expose a web-based terminal interface. This enables agents to execute commands, interact with shells, and provides users with a browser-accessible terminal experience.

Why Does This Exist?

Agents need to:

  • Execute shell commands and capture output
  • Handle terminal interrupts (Ctrl+C) gracefully
  • Maintain terminal state across operations
  • Provide a web-based terminal for remote access

Users need:

  • A browser-accessible terminal to interact with Code Puppy
  • Cross-platform terminal compatibility (Windows, Linux, macOS)
  • Proper handling of terminal edge cases (ANSI codes, interrupts)

How It Works

User Perspective

  1. Start Code Puppy API server with /api_start
  2. Open browser to http://localhost:8765/terminal
  3. Interact with a fully functional terminal via WebSocket
  4. Execute commands, view output, resize terminal

Platform-Specific Handling

Windows

  • Uses pywinpty for PTY emulation
  • Console mode management via ctypes
  • ANSI reset via escape sequences
  • Ctrl+C disabled at console level to prevent terminal bricking

Unix (Linux/macOS)

  • Uses native pty module
  • reset command for terminal restoration
  • Signal handlers for graceful interrupts

Edge Cases & Gotchas

WARNING

Windows + uvx: When launched via uvx, Ctrl+C is disabled at the console level to prevent terminal bricking. Use Ctrl+K to cancel agents instead.

NOTE

Truecolor support: Code Puppy uses 24-bit color. Modern terminals (iTerm2, Windows Terminal, Kitty, Alacritty) recommended. Built-in macOS Terminal.app does not support truecolor.

CAUTION

Terminal reset: After interrupts, terminal state may be corrupted. reset_terminal() performs platform-specific restoration.

Error Scenarios

Scenario Behavior User Sees
PTY creation fails Session not created Error message with platform-specific cause
WebSocket disconnect Session cleaned up Terminal shows "disconnected"
Shell exits Session ends Output shows process exit
Port in use API server fails to start Error: "No available ports in range 8090-9010"

Testing This Feature

Manual Testing

# Start API server
code-puppy /api_start

# In browser, open:
http://localhost:8765/terminal

# Test commands:
echo "Hello"      # Basic output
ls -la            # Directory listing
sleep 5           # Long-running command
Ctrl+C            # Interrupt (or Ctrl+K on Windows+uvx)

Tool Testing

# Check terminal server
result = await check_terminal_server()
assert result["success"] == True

# Open terminal browser
result = await open_terminal_browser()
assert result["success"] == True
