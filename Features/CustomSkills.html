Feature: Custom Skills

What Does This Do?

Custom Skills allow users to create and share modular capability packets that extend Code Puppy's functionality. Think of them as "training manuals" you can dynamically load when needed—like teaching your puppy new tricks on demand!

Skills consist of a SKILL.md file with YAML frontmatter metadata and Markdown instructions, plus optional supporting resource files (templates, examples, configs).

Why Does This Exist?

  • Reusability: Share best practices and workflows across projects
  • Specialization: Give agents domain-specific expertise (DevOps, security, testing, etc.)
  • Extensibility: Extend functionality without modifying core code
  • Community: Enable a skill ecosystem where users can share capabilities

How It Works

User Perspective

  1. Create or install a skill by placing a directory with SKILL.md in a skill directory
  2. Discover skills via /skills list or the interactive TUI menu
  3. Enable/disable skills globally or per-skill via configuration
  4. Activate skills when needed—the agent loads the full instructions and follows them

System Perspective

    participant User
    participant CLI
    participant Plugin
    participant Agent

    User->>CLI: /skills list
    CLI->>Plugin: Discover skills
    Plugin->>Skill: Scan directories
    Skill-->>Plugin: Return metadata
    Plugin-->>CLI: Display skills

    Plugin-->>Agent: Enhanced prompt with skills
    User->>Agent: "Help me with Docker"
    Agent->>Skill: list_or_search_skills(query="docker")
    Skill-->>Agent: Found docker skill
    Agent->>Skill: activate_skill(skill_name="docker")
    Skill-->>Agent: Full SKILL.md content
    Agent->>User: Follows skill instructions

Configuration

Setting Purpose Default
skills_enabled Globally enable/disable skills integration true
skill_directories Directories to scan for skills ["~/.code_puppy/skills", "./skills"]
disabled_skills List of skill names to disable []

Setting Configuration

# Disable skills entirely
/set skills_enabled = false

# Add a custom skill directory
/set skill_directories = "[\"/path/to/skills\"]"

# Disable specific skills
/set disabled_skills = "[\"skill-one\", \"skill-two\"]"

Creating Custom Skills

Required Structure

my-skill/
└── SKILL.md          # Required: Instructions + metadata

Optional Resources

my-skill/
├── SKILL.md              # Required
├── template.py           # Optional resource
├── config.yaml           # Optional resource
└── examples/
    └── sample.json       # Optional resource

SKILL.md Format

---
name: my-expert-skill
description: Expert guidance for specific domain tasks
version: 1.0.0
author: Your Name
tags:
  - domain
  - expertise
---

# Skill Title

## When to Use This Skill

Describe scenarios where this skill applies.

## Instructions

Step-by-step guidance for the agent.

## Best Practices

Key principles to follow.

## Examples

Concrete examples of usage.

Required Frontmatter Fields

Field Type Required Description
name string ✅ Yes Unique skill identifier (kebab-case)
description string ✅ Yes Brief description of what the skill does
version string ❌ No Semantic version (e.g., "1.0.0")
author string ❌ No Author name or email
tags list ❌ No List of keywords for categorization

Edge Cases & Gotchas

WARNING

Skills execute with the same permissions as Code Puppy. Only install skills from trusted sources and review content before use.

NOTE

Skills are discovered at startup. Use /skills refresh to re-scan directories after adding new skills.

CAUTION

Skill names must be unique. If two skills have the same name, only one will be loaded.

Error Scenarios

Scenario Behavior User Sees
Skill directory doesn't exist Skipped during discovery Warning log, skill not listed
SKILL.md missing or invalid Skill not discovered Skill listed but marked as invalid
Frontmatter missing required fields Metadata parse fails Skill not usable, logged error
Skill disabled in config Skipped in prompt/tools Listed as disabled in /skills list
Skills integration disabled All skills ignored "Skills integration is disabled" error

Testing This Feature

Manual Testing Steps

  1. Create a test skill: ```bash mkdir -p ~/.code_puppy/skills/test-skill cat > ~/.code_puppy/skills/test-skill/SKILL.md << 'EOF' --- name: test-skill description: A test skill for verification version: 0.1.0 tags: - test --- # Test Skill When activated, this skill confirms it's working. EOF ```
  1. Refresh discovery: ``` /skills refresh ```
  1. Verify it appears: ``` /skills list ```
  1. Test activation via agent: ``` /agent code-puppy # Ask: "What skills are available?" # Agent should list test-skill ```

Automated Testing

Run the skill-related tests:

pytest tests/plugins/test_agent_skills.py
pytest tests/plugins/test_skill_catalog.py
pytest tests/plugins/test_skill_downloader.py
