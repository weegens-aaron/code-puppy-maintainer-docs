Feature: Model Configuration

What Does This Do?

Model Configuration allows users to customize how each AI model behaves by adjusting parameters like temperature, top-p, seed, reasoning effort, and more. Settings can be configured globally (applies to all models) or per-model (specific to individual models), giving fine-grained control over model behavior.

Why Does This Exist?

Different tasks require different model behaviors:

  • Creative writing needs higher temperature for diversity
  • Code generation needs lower temperature for precision
  • Reproducible results need fixed seed values
  • Cost optimization needs controlled output length via max_tokens

Without configuration, all models would use default settings that may not suit every use case.

How It Works

User Perspective

Users can configure model settings through:

  1. Interactive TUI: /model-settings command opens a beautiful split-panel interface
  2. Slash commands: /set temperature=0.7, /set seed=42, etc.
  3. Config file: Directly edit ~/.code_puppy/puppy.cfg

System Perspective

sequenceDiagram
    participant User
    participant TUI
    participant Config
    participant ModelFactory
    participant API

    User->>TUI: Open model settings
    TUI->>Config: Load all model settings
    Config-->>TUI: Return settings dict
    User->>TUI: Adjust temperature
    TUI->>Config: set_model_setting(model, temp)
    Config->>Config: Write to puppy.cfg
    User->>TUI: Select model
    TUI->>ModelFactory: Create model instance
    ModelFactory->>Config: get_effective_model_settings(model)
    Config-->>ModelFactory: Filtered settings
    ModelFactory->>API: Request with settings

Per-Model Settings

Settings are stored with sanitized model names:

[model_settings_gpt_5_temperature]
value = 0.7

[model_settings_claude_4_0_sonnet_extended_thinking]

Edge Cases & Gotchas

WARNING

Model-specific settings: Not all settings work with all models. The system filters settings based on each model's supported_settings list in models.json.

WARNING

Sanitized keys: Model names are sanitized for config keys (dots, dashes, slashes become underscores). gpt-5.2 becomes gpt_5_2.

NOTE

Fallback chain: Per-model settings → Global settings → Model defaults. If a model doesn't have a setting configured, it falls back to global, then to the model's API default.

CAUTION

Yolo mode: When yolo_mode=true, parallel tool calls are enabled. When yolo_mode=false (default), tool calls are sequential for user review.

Error Scenarios

Scenario Behavior User Sees
Invalid temperature Clamped to 0.0-2.0 Value silently adjusted
Invalid reasoning effort Falls back to medium Warning emitted
Model not in config Uses first available model Error: "No valid model"
Setting not supported Filtered out silently Setting ignored

Testing This Feature

Manual Testing

# Start Code Puppy
code-puppy

# Open model settings TUI
/model-settings

# Or use slash commands
/set temperature=0.8
/set seed=42
/set openai_reasoning_effort=high

# Verify settings persisted
cat ~/.code_puppy/puppy.cfg | grep model_settings

Programmatic Testing

from code_puppy.config import (
    get_all_model_settings,
    set_model_setting,
    get_effective_model_settings,
)

# Set a per-model setting
set_model_setting("gpt-5", "temperature", 0.7)

# Get all settings for a model
settings = get_all_model_settings("gpt-5")
# Returns: {"temperature": 0.7, ...}

# Get effective settings (with fallbacks)
effective = get_effective_model_settings("gpt-5")
# Returns filtered settings ready for API
