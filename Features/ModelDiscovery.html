Feature: Model Discovery

What Does This Do?

Model Discovery allows users to browse, search, and add AI models from a comprehensive database of hundreds of models across dozens of providers - all directly from the terminal UI. It fetches live data from models.dev API (with bundled fallback) and provides an interactive interface to add models to your configuration.

Why Does This Exist?

Maintaining a list of available AI models manually is tedious. Code Puppy provides:

  • Centralized database: Access to models.dev's curated model registry
  • Interactive browsing: Terminal UI to explore providers and models
  • One-click addition: Add models to your config without manual JSON editing
  • Capability filtering: Find models by vision, reasoning, tool use, etc.
  • Cost awareness: See input/output costs before adding

How It Works

User Perspective

  1. Run /add-model command in the terminal
  2. Browse providers in the left panel (paginated)
  3. Select a provider to see its models in the right panel
  4. Select a model to see detailed information (cost, context, capabilities)
  5. Press Enter to add the model to extra_models.json
  6. Provide API credentials if prompted

Edge Cases & Gotchas

WARNING

Provider support varies: Some providers (AWS Bedrock, Google Vertex, Cloudflare) require non-OpenAI-compatible authentication and are marked as unsupported in the UI.

NOTE

Fallback behavior: If models.dev API times out or fails, the system automatically falls back to a bundled JSON database (models_dev_api.json ~535KB).

TIP

Custom models: The UI includes a "✨ Custom model..." option at the bottom of each provider's model list for adding models not in the database.

Error Scenarios

Scenario Behavior User Sees
models.dev API timeout Falls back to bundled DB Warning: "models.dev API timed out, using bundled fallback"
Invalid JSON in bundled file Raises FileNotFoundError Error: "No data source available"
Missing API key for selected model Skips model creation Warning: "API key is not set for [provider] endpoint"
Provider not supported Shows ⚠️ icon Dimmed text with warning icon
