Feature: MCP Catalog

What Does This Do?

The MCP Catalog is a curated collection of pre-configured MCP servers that can be easily searched, browsed, and installed through the Code Puppy CLI. Instead of manually configuring each server, users can discover and install servers with a few commands.

Why Does This Exist?

MCP (Model Context Protocol) servers extend Code Puppy's capabilities (filesystem access, database queries, browser control, etc.), but configuring them requires understanding JSON schemas, environment variables, and command-line arguments. The catalog:

  • Reduces friction for new users discovering MCP servers
  • Provides verified configurations that are known to work
  • Categorizes servers by use case (Database, Web, DevOps, etc.)
  • Handles complexity by managing environment variables and dependencies

How It Works

User Perspective

  1. Browse available servers via /mcp search <query> or /mcp list
  2. View server details including requirements and configuration
  3. Install with /mcp install <server-id>
  4. The system prompts for any required environment variables or configuration values
  5. Server is automatically added to mcp_servers.json and can be started

Configuration

Server Template Structure

Each server in the catalog is defined as an MCPServerTemplate:

MCPServerTemplate(
    id="postgres",
    name="postgres",
    display_name="PostgreSQL Database",
    description="Connect to and query PostgreSQL databases",
    category="Database",
    tags=["database", "sql", "postgres"],
    type="stdio",  # or "http", "sse"
    config={
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-postgres", "${connection_string}"],
    },
    requires=MCPServerRequirements(
        environment_vars=["DATABASE_URL"],
        command_line_args=[{
            "name": "connection_string",
            "prompt": "PostgreSQL connection string",
            "default": "postgresql://localhost/mydb",
            "required": True,
        }],
        required_tools=["node", "npm", "npx"],
        package_dependencies=["@modelcontextprotocol/server-postgres"],
    ),
    verified=True,
    popular=True,
)

Available Hooks

Plugins can extend the catalog via the register_mcp_catalog_servers callback:

from code_puppy.callbacks import register_callback

def _on_register_mcp_catalog_servers():
    from code_puppy.mcp_.server_registry_catalog import MCPServerTemplate, MCPServerRequirements
    
    return [
        MCPServerTemplate(
            id="my-custom-server",
            name="my-custom-server",
            display_name="My Custom Server",
            description="A custom MCP server for...",
            category="Custom",
            tags=["custom", "special"],
            type="stdio",
            config={"command": "my-command", "args": []},
            requires=MCPServerRequirements(
                required_tools=["my-tool"],
            ),
        )
    ]

register_callback("register_mcp_catalog_servers", _on_register_mcp_catalog_servers)

Edge Cases & Gotchas

WARNING

Placeholders in config: Use ${VAR_NAME} syntax for values that should be prompted at install time (e.g., connection strings, paths).

NOTE

Plugin extension: The catalog is built at import time. Plugins adding servers via callback will have them included, but the global catalog instance is created before plugins load. Use MCPServerCatalog() to get a fresh instance with plugin servers.

CAUTION

Verification flag: verified=True indicates the server configuration has been tested. Users should still verify API keys and permissions work in their environment.

Error Scenarios

Scenario Behavior User Sees
Server not found in catalog Returns empty search results "No servers matching 'xyz'"
Missing required tool Installation menu shows warning "Requires: node, npm (not found)"
Invalid environment variable Server config written but won't start Server fails to start, check logs
Duplicate server name Registry handles by name "Server 'postgres' already exists"
