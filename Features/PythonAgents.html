Feature: Python Agents

What Does This Do?

Python Agents are specialized AI agents with deep expertise in Python development, code review, and modern Python best practices. They provide domain-specific intelligence for Python projects, understanding everything from type hints and async patterns to data science workflows and web framework idioms.

Why Does This Exist?

While general-purpose agents can handle many tasks, Python-specific agents bring focused expertise:

  • Pythonic Code Quality: They enforce PEP 8, PEP 20, and modern Python idioms
  • Type Safety: They demand complete type annotations and mypy compliance
  • Async Mastery: They understand asyncio patterns and concurrency best practices
  • Ecosystem Knowledge: They know the Python ecosystem (FastAPI, Django, Pandas, NumPy, etc.)
  • Tooling Integration: They leverage Python-specific tools (pytest, ruff, black, bandit)

How It Works

User Perspective

  1. User selects a Python agent from the agent menu (/agent)
  2. Agent introduces itself with Python-specific personality
  3. User provides Python-related tasks (code generation, review, debugging)
  4. Agent applies Python expertise to solve problems
  5. Agent validates work using Python tooling (mypy, pytest, ruff)

Edge Cases & Gotchas

WARNING

Python agents are hardcoded classes - Unlike JSON agents, Python agents are defined as Python classes in code_puppy/agents/. To create a new Python agent, you must add a new class file and register it.

NOTE

Python agents have specialized prompts - Each Python agent has a carefully crafted system prompt (~1500 tokens) that defines their expertise and personality. These prompts are much more detailed than generic agent prompts.

TIP

Use python-reviewer for PRs - The python-reviewer agent is specifically designed for pull request reviews. It focuses on meaningful diffs and provides actionable feedback with exact line references.

CAUTION

Python programmer writes code - The python-programmer agent will create and modify files. Always review changes before committing, especially for complex refactors.

Python-Specific Workflows

Code Generation Workflow

  1. User describes Python feature or function
  2. Agent analyzes existing codebase patterns
  3. Agent writes type-annotated, PEP 8 compliant code
  4. Agent runs mypy and ruff for validation
  5. Agent creates file with proper docstrings
  6. Agent suggests tests

Code Review Workflow

  1. User invokes python-reviewer with code changes
  2. Agent reads modified files
  3. Agent checks for issues by severity (blockers → warnings → nits)
  4. Agent provides exact line references
  5. Agent groups related issues
  6. Agent gives final verdict (Ship it / Needs fixes / Mixed bag)
