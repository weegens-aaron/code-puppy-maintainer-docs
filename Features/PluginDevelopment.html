Feature: Plugin Development

What Does This Do?

Plugins allow you to extend Code Puppy's functionality without modifying core code. They can add custom commands, tools, agents, model types, and hook into every stage of the system's operation through a comprehensive callback system.

Why Does This Exist?

Code Puppy is designed to be highly extensible. Instead of forking the project or editing core files, you can:

  • Add custom slash commands (e.g., /mycommand)
  • Register new tools for agents to use
  • Create custom agents
  • Add new model types (e.g., custom LLM providers)
  • Hook into system events (startup, tool calls, streaming, etc.)
  • Inject custom content into system prompts

This keeps core stable while enabling unlimited customization.

How It Works

User Perspective

  1. Create a plugin directory at ~/.code_puppy/plugins/my_plugin/
  2. Add a register_callbacks.py file
  3. Register callbacks for the hooks you want to use
  4. Code Puppy auto-discovers and loads the plugin on startup

System Perspective

flowchart TD
    A[Code Puppy Startup] --> B[Load Builtin Plugins]
    B --> C[Load User Plugins]
    C --> D[Plugins Register Callbacks]
    D --> E[Callbacks Stored in Registry]
    E --> F[System Events Trigger Callbacks]
    F --> G[Plugin Code Executes]

Available Hooks

See [code_puppy/callbacks.py](../code_puppy/callbacks.py) for the complete list. Key hooks include:

Hook When Use Case
startup App boot Initialize resources, log startup
shutdown Graceful exit Cleanup, save state
custom_command Unknown /slash cmd Add custom commands
custom_command_help /help menu Add help entries
register_tools Tool registration Add new agent tools
register_agents Agent catalogue Add custom agents
register_model_type Model creation Add custom model types
pre_tool_call Before tool executes Log, validate, modify
post_tool_call After tool finishes Log results, metrics
stream_event Response streaming Real-time event processing
agent_run_start Agent task begins Setup, tracking
agent_run_end Agent task ends Cleanup, analytics
get_model_system_prompt Prompt assembly Inject custom content

Configuration

Plugins read configuration from:

  • ~/.code_puppy/config.json - Global config
  • Plugin-specific config files in plugin directory

Example config injection:

from code_puppy.config import get_config

def my_handler():
    config = get_config()
    my_setting = config.get("my_plugin", {}).get("setting", "default")

Edge Cases & Gotchas

WARNING

Callbacks are deduplicated - The same callback function registered twice will only fire once. This prevents duplicate registrations if plugins are accidentally loaded multiple times.

WARNING

**Return None for unhandled commands** - In custom_command handlers, return None if you don't recognize the command. Return True if you handled it (no further action needed).

NOTE

Sync vs Async - Callbacks can be sync or async. The system handles both. Async callbacks receive await automatically.

NOTE

Plugin loading is idempotent - Plugins are only loaded once per session. Subsequent calls to load_plugin_callbacks() return empty lists.

Error Scenarios

Scenario Behavior User Sees
Plugin raises exception on load Logged as error, plugin skipped Warning in logs
Callback raises exception Logged as error, other callbacks continue Error in logs
Invalid hook name ValueError raised during registration Python error

Testing This Feature

Manual Testing

  1. Create a test plugin at ~/.code_puppy/plugins/test_plugin/register_callbacks.py
  2. Add a simple callback: ```python from code_puppy.callbacks import register_callback def _on_startup(): print("TEST PLUGIN LOADED!") register_callback("startup", _on_startup) ```
  3. Restart Code Puppy
  4. Check logs for "TEST PLUGIN LOADED!"

Programmatic Testing

from code_puppy.callbacks import count_callbacks, get_callbacks

# Check how many callbacks registered for a hook
count = count_callbacks("startup")
print(f"Startup callbacks: {count}")

# Get specific callbacks
callbacks = get_callbacks("custom_command")
print(f"Custom command handlers: {len(callbacks)}")

Examples

Example 1: Simple Custom Command

from code_puppy.callbacks import register_callback
from code_puppy.messaging import emit_info

def _custom_help():
    return [("hello", "Say hello to Code Puppy")]

def _handle_command(command: str, name: str):
    if name == "hello":
        emit_info("👋 Hello! How can I help?")
        return True
    return None

register_callback("custom_command_help", _custom_help)
register_callback("custom_command", _handle_command)

Example 2: Tool Interceptor

import logging
from code_puppy.callbacks import register_callback

logger = logging.getLogger(__name__)

async def _log_tool_call(tool_name: str, tool_args: dict, context=None):
    logger.info(f"Tool called: {tool_name} with args: {tool_args}")

register_callback("pre_tool_call", _log_tool_call)

Example 3: Custom Model Type

from code_puppy.callbacks import register_callback

def create_my_custom_model(model_name: str, model_config: dict, config: dict):
    # Create and return your custom model instance
    # Return None if this doesn't handle the model
    if model_name.startswith("my-custom-"):
        return MyCustomModel(model_name, model_config)
    return None

def _register_model_types():
    return [{"type": "my-custom", "handler": create_my_custom_model}]

register_callback("register_model_type", _register_model_types)

Example 4: Agent Lifecycle Hooks

from code_puppy.callbacks import register_callback
from code_puppy.messaging import emit_success, emit_info

async def _on_agent_start(agent_name: str, model_name: str, session_id=None):
    emit_info(f"🚀 Agent '{agent_name}' starting with model '{model_name}'")

async def _on_agent_end(agent_name: str, model_name: str, success=True, error=None, **kwargs):
    if success:
        emit_success(f"✅ Agent '{agent_name}' completed successfully")
    else:
        emit_info(f"❌ Agent '{agent_name}' failed: {error}")

register_callback("agent_run_start", _on_agent_start)
register_callback("agent_run_end", _on_agent_end)

Plugin Development Workflow

  1. Plan - Decide what hooks you need
  2. Create - Make plugin directory and register_callbacks.py
  3. Implement - Write callback functions
  4. Test - Restart Code Puppy, verify behavior
  5. Document - Add README.md with usage instructions
  6. Distribute - Share via git repo or local install

Best Practices

IMPORTANT

1. Keep it simple - One plugin, one responsibility 2. Fail gracefully - Wrap callbacks in try/except 3. Log appropriately - Use logging module, not print() 4. **Return None - For callbacks you don't handle 5. Document** - Add README.md explaining usage

Plugin Discovery

Code Puppy discovers plugins by:

  1. Scanning code_puppy/plugins/ for builtin plugins
  2. Scanning ~/.code_puppy/plugins/ for user plugins
  3. Looking for directories containing register_callbacks.py
  4. Importing each plugin module at startup

Plugins without register_callbacks.py are ignored.
