Feature: Skill Installation

What Does This Do?

Skill installation allows users to browse and install remote agent skills from a centralized catalog, as well as manage locally created skills. Skills are reusable capabilities that extend what Code Puppy's AI agents can do - from file operations to browser automation to domain-specific expertise.

Why Does This Exist?

  • Extensibility: Users can add new capabilities without modifying core code
  • Reusability: Skills can be shared across projects and users
  • Community: Remote catalog enables community-contributed skills
  • Modularity: Skills are self-contained with their own documentation (SKILL.md)

How It Works

User Perspective

  1. Run /skills install to browse the remote catalog
  2. Navigate categories and select a skill
  3. Confirm installation - the skill downloads as a ZIP and extracts to ~/.code_puppy/skills/
  4. The skill becomes available for agents to use (if skills integration is enabled)

Installation Flow Details

1. Catalog Fetch

  • Source: https://www.llmspec.dev/skills/skills.json
  • Caching: ~/.code_puppy/cache/skills_catalog.json (30min TTL)
  • Offline fallback: Uses stale cache if network fails
  • Error handling: Never crashes app - shows empty catalog if unavailable

2. ZIP Download

  • Streaming: Downloads in chunks to avoid memory issues
  • Timeout: 30 seconds
  • User-Agent: code-puppy/skill-downloader

3. Safety Validation

# Zip-slip protection
- Rejects absolute paths (e.g., `/etc/passwd`)
- Rejects parent traversal (e.g., `../../evil`)

# Zip bomb protection
- Max uncompressed size: 50MB
- Validates each entry size

# Layout validation
- Must contain SKILL.md at root or single subdirectory
- Skips `__MACOSX` metadata folders

4. Normalized Installation

  • Extracts to temp directory first
  • Detects whether files are at root or in single subdirectory
  • Copies to ~/.code_puppy/skills/<skill_name>/
  • Verifies SKILL.md exists after install

Edge Cases & Gotchas

WARNING

Reinstall requires confirmation - If a skill is already installed, you must explicitly confirm y to reinstall.

NOTE

Catalog may be empty - If network is unavailable AND no cache exists, the catalog will be empty. Check ~/.code_puppy/cache/ for cached data.

CAUTION

Skill name must be simple - Cannot contain path separators or special characters. The downloader validates Path(skill_name).name == skill_name.

Error Scenarios

Scenario Behavior User Sees
Network timeout Falls back to cache "Remote fetch failed; falling back to cached catalog"
Invalid ZIP Installation rejected "Rejected unsafe zip: [reason]"
Missing SKILL.md Installation rejected "Extracted zip missing SKILL.md"
Disk write failure Cleanup partial install "Failed to move skill into place"
Already installed Prompts for reinstall "Skill already installed. Reinstall? [y/N]"

Usage Examples

Browse and Install from Catalog

$ code_puppy
> /skills install

# Navigate with ↑/↓, Enter to select category, Enter to install
# Press Ctrl+C to cancel

List Local Skills

> /skills list

✓ data-exploration v1.0 by Code Puppy
    tags: data, analysis
✓ browser-control v2.1
    tags: browser, automation
✗ disabled custom-skill

Manage Skill Directories

> /skills
# Press Ctrl+A to add a directory
# Press Ctrl+D to show/manage directories

Enable/Disable Skills Integration

> /skills enable    # Enable globally
> /skills disable   # Disable globally
> /skills toggle    # Toggle current state

Testing This Feature

Manual Test

  1. Run code_puppy
  2. Type /skills install
  3. Verify catalog loads (or shows cached data)
  4. Navigate to a category and select a skill
  5. Confirm installation
  6. Verify skill appears in ~/.code_puppy/skills/
  7. Run /skills list to confirm it's detected

Verify Installation

# Check directory exists
ls ~/.code_puppy/skills/<skill-name>/

# Verify SKILL.md present
cat ~/.code_puppy/skills/<skill-name>/SKILL.md

# Check cache
ls ~/.code_puppy/cache/skills_catalog.json
