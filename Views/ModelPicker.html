View: Model Picker

Overview

Property Value
Route /model (slash command)
Auth Required N/A (local CLI)
Layout Single panel with autocomplete

What Does This View Do?

The Model Picker allows users to switch the active AI model used by the current agent. It provides both a command-line completion interface (typing /model <model-name>) and can be invoked via the /model slash command. The picker shows available models with metadata indicating which is currently active.

User Actions

Action Trigger Result
Type model name /model <partial-name> Shows matching models
Select model Tab / Enter Switches to selected model
Cancel Esc Keeps current model

Edge Cases & Gotchas

NOTE

Trigger requirement: The completion only triggers when /model (with space) is at the beginning of the line.

TIP

Case-insensitive: Model name matching is case-insensitive for convenience.

WARNING

Model validation: Invalid model names are not shown in completions, preventing errors.
