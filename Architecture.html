System Overview

What Is This?

Code Puppy is an AI-powered code generation agent that provides a terminal-based interactive interface for working with multiple AI models. It enables users to create custom agents, manage MCP servers, install skills, and execute complex coding tasks through a sassy, personality-driven CLI experience. Unlike expensive IDEs, Code Puppy runs locally and supports 65+ AI providers with full privacy control.

Quick Start for Maintainers

# Run interactively
uvx code-puppy -i

# Run non-interactively
uvx code-puppy "fix the bug in my code"

# Run tests
pytest tests/ -v

# Check logs
~/.code_puppy/logs/

Plugin Architecture

Code Puppy uses a callback-based plugin system:

sequenceDiagram
    participant User
    participant Core
    participant HookEngine
    participant Plugins
    
    User->>Core: Action (e.g., tool call)
    Core->>HookEngine: Trigger hook
    HookEngine->>Plugins: Call registered callbacks
    Plugins-->>HookEngine: Results
    HookEngine-->>Core: Aggregated result
    Core-->>User: Final response

Available Hooks

Hook When Use Case
startup App boot Initialize plugins
shutdown Exit Cleanup resources
invoke_agent Agent starts Pre-processing
agent_run_start/end Agent lifecycle Monitoring, workflows
pre_tool_call/post_tool_call Tool execution Logging, validation
run_shell_command Shell exec Safety checks
file_permission File ops Access control
custom_command Unknown slash cmd Custom handlers
register_tools Tool registration Plugin tools
register_agents Agent discovery Plugin agents

See code_puppy/callbacks.py for full hook list.

Security Model

  • Local-first: All data stays on user machine
  • No telemetry: Zero analytics or data collection
  • API keys: Stored in environment variables
  • File permissions: Optional permission checks via hooks
  • Shell safety: Command validation and blocking hooks
